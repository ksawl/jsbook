## |1| **Доступ к элементам формы**

В JavaScript есть специальный способ быстрого доступа к формам и их полям через свойство `document.forms`.

```javascript
const form = document.forms.loginForm; // Доступ по атрибуту name
const emailInput = form.elements.email; // Доступ к полю по name
```

Основные события форм:
-   `submit` — происходит при попытке отправить форму.
-   `input` — происходит при каждом изменении значения в поле.
-   `change` — происходит, когда поле теряет фокус после изменения.

### **Атрибут `target` в формах**

Атрибут `target` указывает, где именно браузер должен открыть результат отправки формы (например, ответ сервера).

-   **`_self`** (по умолчанию): Открывает результат в той же вкладке.
-   **`_blank`**: Открывает результат в новой вкладке.
-   **`_parent`** и **`_top`**: Используются при работе с фреймами (`iframe`) для открытия результата в родительском или самом верхнем окне.
-   **Имя фрейма**: Если на странице есть `<iframe name="myFrame">`, то `target="myFrame"` отправит результат формы прямо внутрь этого фрейма.

## |2| **Получение данных (FormData)**

Вместо того чтобы вручную собирать `input.value` для каждого поля, лучше использовать объект `FormData`. Он автоматически собирает все данные формы, где указан атрибут `name`.

```javascript
form.addEventListener('submit', (e) => {
    e.preventDefault();
    const data = new FormData(form);
    
    console.log(data.get('email')); // Получить конкретное поле
    
    // Превратить в обычный объект
    const obj = Object.fromEntries(data.entries());
    console.log(obj);
});
```

## |3| **Валидация: HTML5 vs JavaScript**

### **1. Встроенная (HTML5)**
Используйте атрибуты `required`, `type="email"`, `pattern` (регулярные выражения), `minlength`. Браузер сам не даст отправить форму, если правила нарушены.

### **2. Программная (JavaScript)**
Позволяет делать сложные проверки (например, "пароли совпадают").

```javascript
if (password.value.length < 8) {
    alert('Пароль слишком короткий!');
    return;
}
```

## |4| **Основы отправки данных (Fetch API)**

После того как данные собраны и проверены, их нужно отправить на сервер. `Fetch` — это современный способ делать сетевые запросы без перезагрузки страницы.

```javascript
fetch('/api/login', {
    method: 'POST',
    body: JSON.stringify(obj),
    headers: {
        'Content-Type': 'application/json'
    }
})
.then(res => res.json())
.then(result => console.log('Ответ сервера:', result))
.catch(err => console.error('Ошибка:', err));
```

---

## **Итог**

-   Используйте `document.forms` и `form.elements` для удобного доступа.
-   Всегда делайте `event.preventDefault()` в обработчике `submit`.
-   **FormData** — лучший способ собрать данные из всех полей разом.
-   **HTML5 валидация** — быстрая и надежная база, **JS валидация** — для сложной логики.

## **Практика**

### 1. **Живая валидация**
Создайте форму регистрации с полем "Пароль" и "Повторите пароль". Сделайте так, чтобы при вводе во второе поле, если пароли не совпадают, граница поля становилась красной.

### 2. **Сборщик анкеты**
Создайте большую форму (Имя, Возраст, Хобби через checkbox, Пол через radio). При отправке формы соберите данные через `FormData` и выведите в консоль итоговый объект.

### 3. **Мини-регистрация (MOCK)**
Реализуйте отправку данных формы на фейковый адрес `https://jsonplaceholder.typicode.com/posts` через `fetch`. Выведите ответ сервера на страницу.

### 4. **Умная кнопка**
Сделайте кнопку отправки формы заблокированной (`disabled`), пока не заполнены все обязательные поля (используйте событие `input` для проверки всей формы).
