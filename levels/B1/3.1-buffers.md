## |1| **Что такое Buffer?**

**Buffer** — это область памяти, выделенная за пределами обычного "движка" V8. Она имеет фиксированный размер и предназначена для хранения бинарных данных.

### **Зачем это нужно?**
Бинарные данные встречаются повсеместно:
-   Чтение файлов с картинками или видео.
-   Прием данных по сети.
-   Шифрование и хеширование.

Буферы в Node.js ведут себя похожим образом на массивы, но их размер нельзя изменить после создания. Каждая ячейка буфера хранит один байт (целое число от 0 до 255).

---

## |2| **Создание буферов**

В современных версиях Node.js создание буферов через `new Buffer()` считается устаревшим и небезопасным. Используйте статические методы:

### **1. Buffer.alloc(size)**
Создает "чистый" буфер заданного размера, заполненный нулями.
```javascript
const buf = Buffer.alloc(10); // 10 байт, заполненных нулями
console.log(buf); // <Buffer 00 00 00 00 00 00 00 00 00 00>
```

### **2. Buffer.from(data)**
Создает буфер на основе существующих данных (строки, массива или другого буфера).
```javascript
const bufFromStr = Buffer.from('Hello');
console.log(bufFromStr); // <Buffer 48 65 6c 6c 6f> (ASCII коды символов)

const bufFromArr = Buffer.from([1, 2, 3]);
console.log(bufFromArr); // <Buffer 01 02 03>
```

---

## |3| **Чтение и запись**

Мы можем обращаться к буферу по индексам, как к массиву.

### **Чтение по индексу**
```javascript
const buf = Buffer.from('Hey');
console.log(buf[0]); // 72 (ASCII код буквы 'H')
```

### **Запись по индексу**
```javascript
const buf = Buffer.alloc(3);
buf[0] = 72;
buf[1] = 105;
console.log(buf.toString()); // "Hi"
```

## |4| **Кодировки (UTF-8, Base64, Hex)**

Буфер сам по себе не знает, что за данные в нем лежат. Мы интерпретируем их с помощью кодировок при выводе.

-   **UTF-8**: Стандарт для текста.
-   **Hex**: Шестнадцатеричное представление (часто для ключей и хешей).
-   **Base64**: Способ записи бинарных данных строкой (например, для передачи картинок в теле HTML).

```javascript
const buf = Buffer.from('Node.js', 'utf8');

console.log(buf.toString('hex')); // 4e6f64652e6a73
console.log(buf.toString('base64')); // Tm9kZS5qcw==
```

---

## **Итог**

-   **Buffer** — это "массив байтов" фиксированной длины.
-   Используется для работы с файлами, потоками и сетевыми пакетами.
-   **Buffer.alloc** — выделение пустой памяти.
-   **Buffer.from** — импорт данных в бинарный вид.
-   **toString()** позволяет превратить байты обратно в понятный текст или строку.

## **Практика**

### 1. **Ручная кодировка**
Создайте буфер из 5 байт. Запишите в него коды символов для слова `Apple` вручную (используйте таблицу ASCII) и выведите результат через `toString()`.

### 2. **Base64 Конвертер**
Напишите функцию `textToBase64(text)`, которая принимает любую строку и возвращает её в формате Base64, используя буферы.

### 3. **Тайное послание (Hex)**
Дан буфер: `<Buffer 4a 53 20 69 73 20 61 77 65 73 6f 6d 65>`.
Создайте его в коде через `Buffer.from(..., 'hex')` и расшифруйте, что там написано.

### 4. **Сравнение буферов**
Создайте два буфера с одинаковым текстом, но один из них через `Buffer.from`, а другой через `Buffer.alloc` + ручное заполнение. Используйте метод `.equals()`, чтобы проверить их идентичность.
