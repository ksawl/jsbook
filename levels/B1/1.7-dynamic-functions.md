# **Динамические функции**

В JavaScript функции — это не просто блоки кода, а объекты. Мы можем не только объявлять их заранее, но и создавать прямо во время выполнения программы из обычных строк. Именно об этой "магии" — конструкторе `new Function` — мы и поговорим.

## |1| **Конструктор new Function**

Обычно мы создаем функции через `function declaration` или `arrow functions`. Однако есть третий путь: создание функции "на лету" из строки.

### **Синтаксис**
```javascript
let func = new Function([arg1, arg2, ...argN], functionBody);
```
-   `arg1...argN` — имена будущих аргументов (в виде строк).
-   `functionBody` — тело функции (тоже строка).

### **Пример**
```javascript
let sum = new Function('a', 'b', 'return a + b');

console.log(sum(2, 3)); // 5
```

Мы можем даже не передавать аргументы, если они не нужны:
```javascript
let sayHi = new Function('console.log("Привет с того света (строки)!")');
sayHi();
```

## |2| **Особенности и ограничения**

### **1. Область видимости (Closures)**
Это самое важное отличие `new Function`. Обычные функции помнят "родительское" окружение (замыкание). Функции, созданные через `new Function`, **не имеют доступа к локальным переменным**. Они видят только **глобальный объект** и свои аргументы.

```javascript
let value = "Global";

function getFunc() {
    let value = "Local";
    let func = new Function('console.log(value)'); // Смотрит в Global
    return func;
}

getFunc()(); // Выведет "Global", а не "Local"
```

### **2. Безопасность и минификация**
Обычно код JavaScript сжимается (минифицируется) перед отправкой в браузер. Переменные типа `let userName` превращаются в `let a`. Но `new Function` ожидает строку с *оригинальными* именами. Это может сломать код, если вы пытаетесь обратиться к переменным вне функции.

---

## |3| **Зачем это нужно (Use Cases)**

В реальной разработке этот метод используется редко, но он незаменим в специфических задачах:

1.  **Шаблонизаторы**: Когда вы пишете свой движок, который превращает строку типа `Hello, {name}` в исполняемый код.
2.  **Динамическая логика**: Если конфигурация поведения приходит с сервера в виде строки кода (крайне осторожно!).
3.  **Математические парсеры**: Создание функций для вычисления сложных формул, введенных пользователем.

---

## |4| **Сравнение с eval**

Существует еще функция `eval()`, которая тоже исполняет строки. Но между ними есть огромная разница:

| Характеристика | `new Function` | `eval()` |
| :--- | :--- | :--- |
| **Доступ к локальным переменным** | Нет | Да (весь контекст) |
| **Безопасность** | Выше (изолирована) | Очень низкая |
| **Оптимизация движком** | Оптимизируется лучше | Почти не оптимизируется |

> [!WARNING]
> Главное правило работы с `new Function` и `eval`: **Никогда не выполняйте строку, полученную от пользователя**. Это прямой путь к XSS-атакам и взлому вашего приложения.

---

## **Итог**

-   `new Function` позволяет создавать функции из строк.
-   Главная особенность: **отсутствие доступа к локальным замыканиям** (видит только глобальные переменные).
-   Используйте этот метод только тогда, когда код функции действительно генерируется динамически (например, в шаблонизаторах).

## **Практика**

### 1. **Динамический Калькулятор**
Напишите функцию `createCalculator(operator)`, которая принимает строку (`'+'`, `'-'`, `'*'`, `'/'`) и возвращает новую функцию, созданную через `new Function`, которая выполняет соответствующее действие над двумя числами.

```javascript
const add = createCalculator('+');
console.log(add(5, 5)); // 10
```

### 2. **Мини-Шаблонизатор**
Создайте функцию `render(template, data)`, которая принимает строку шаблона (например, `"return 'Привет, ' + name"`) и объект с данными. Функция должна создать динамическую функцию, передать в неё ключи объекта как аргументы и вернуть результат.

### 3. **Глобальный счетчик**
Проверьте на практике: создайте локальную переменную `counter` внутри функции, а затем создайте динамическую функцию через `new Function`, которая пытается её изменить. Опишите результат и причину такого поведения.
