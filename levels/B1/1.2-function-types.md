# **Виды функций: Анонимные, Стрелочные, IIFE**

## |1| **Анонимные функции**

Анонимные функции — это функции без имени. Они не могут существовать в коде «сами по себе» и должны быть либо присвоены переменной, либо переданы как аргумент (колбэк).

---

### **Преимущества и недостатки**

| Критерий | Характеристика |
| :--- | :--- |
| **Краткость** | Позволяют не придумывать имена для «одноразовых» функций. |
| **Изоляция** | Часто используются в IIFE для создания локальной области видимости. |
| **Отладка** | **Минус:** В Stack Trace отображаются как `(anonymous)`, что затрудняет поиск ошибок. |
| **Повтор** | **Минус:** К ним нельзя обратиться повторно, если они не сохранены в переменную. |

---

## |2| **Стрелочные функции**

Стрелочные функции (**Arrow Functions**) — самый компактный синтаксис, введенный в ES6.

### **Синтаксические правила**
1. **Один аргумент**: Можно опустить скобки: `n => n * 2`.
2. **Без аргументов**: Пустые скобки обязательны: `() => console.log('Hi')`.
3. **Одна строка**: Результат возвращается автоматически (неявный `return`).
4. **Фигурные скобки**: Если тело функции многострочное, `return` обязателен.

### **Фундаментальные отличия**
- **Лексический `this`**: Не имеют своего контекста.
- **Нет `arguments`**: Не имеют доступа к псевдомассиву аргументов внешней функции.
- **Не конструкторы**: Нельзя вызвать через `new`.

## |3| **IIFE (Immediately Invoked Function Expression)**

Это функции, которые выполняются немедленно в месте их создания.

### **Зачем это нужно?**
1. **Инкапсуляция**: Скрытие переменных от глобальной области видимости.
2. **Инициализация**: Выполнение кода настройки один раз при загрузке скрипта.
3. **Модульность**: Использовались как основа паттерна «Модуль» до появления ESM.

```javascript
const app = (function() {
  let privateCounter = 0; // Скрыто от всех
  return {
    increment() { privateCounter++; },
    get() { return privateCounter; }
  };
})();
```

## |4| **Сравнение видов функций**

| Характеристика | Declaration | Expression | Arrow |
| :--- | :--- | :--- | :--- |
| **Поднятие (Hoisting)** | Да | Нет | Нет |
| **Свой this** | Да | Да | Нет |
| **Псевдомассив arguments** | Да | Да | Нет |
| **Имя функции** | Обязательно | Опционально | Нет |

---

## **Итог**

- Функции — это «объекты первого класса»: их можно сохранять, передавать и возвращать.
- Выбирайте **Arrow** для колбэков и методов, где важен лексический контекст.
- Используйте **Declaration**, если хотите вызывать функцию до её объявления в коде.

---

## **Практика**

### 1. **Динамическое приветствие**
Создайте стрелочную функцию `welcome`, которая принимает возраст и возвращает однострочное условие: `'Привет!'`, если меньше 18, иначе `'Здравствуйте!'`.

### 2. **Безопасный счетчик (IIFE)**
Реализуйте IIFE, которая возвращает объект с двумя методами: `up()` для увеличения внутренней переменной и `down()` для уменьшения. Переменная должна быть недоступна напрямую.

### 3. **Трассировка**
Создайте анонимную функцию внутри `setTimeout`, которая выбрасывает ошибку. Посмотрите в консоли браузера, как она отображается в стек-трейсе.
