## |1| **Что такое SASS/SCSS?**

**SASS** — это "прокачанный" CSS. Он позволяет использовать переменные, вложенные правила, миксины (функции для CSS) и математические операции.

Существует два синтаксиса:
-   **SASS**: Старый синтаксис (без скобок и точек с запятой).
-   **SCSS**: Современный синтаксис, максимально похожий на обычный CSS. Мы будем использовать его.

Браузеры не понимают SCSS напрямую, поэтому его нужно **компилировать** в обычный CSS.

---

## |2| **Основные возможности SCSS**

### **1. Переменные**
В SASS переменные начинаются с `$`.
```scss
$primary-color: #3498db;
$font-stack: Helvetica, sans-serif;

body {
  color: $primary-color;
  font-family: $font-stack;
}
```

### **2. Вложенность (Nesting)**
Больше не нужно писать длинные селекторы типа `.nav .nav-item .link`.
```scss
.nav {
  background: white;
  .nav-item {
    display: inline-block;
    a {
      color: blue;
      &:hover { color: red; } // & ссылается на родителя (a)
    }
  }
}
```

### **3. Миксины (Mixins)**
Это переиспользуемые блоки кода.
```scss
@mixin flex-center {
  display: flex;
  justify-content: center;
  align-items: center;
}

.header {
  @include flex-center;
}
```

## |3| **Автоматизация сборки (Gulp & Webpack)**

Когда проект становится большим, ручная компиляция SASS превращается в мучение. Для автоматизации этого процесса используют сборщики.

-   **Gulp**: Инструмент для автоматизации рутины (компиляция SASS, минификация картинок, обновление страницы). Работает как "конвейер".
-   **Webpack**: Мощный сборщик модулей. Он анализирует ваш проект и склеивает все файлы (JS, CSS, картинки) в один "бандл" для быстрой загрузки.

---

## **Итог**

-   **SCSS** делает CSS чище и мощнее благодаря переменным и вложенности.
-   **Миксины** позволяют избегать дублирования кода.
-   **Сборщики** (Gulp/Webpack) берут на себя всю грязную работу по подготовке проекта к публикации.

## **Практика**

### 1. **Переменные и Вложенность**
Превратите обычный CSS код в SCSS, используя переменные для цветов и вложенный синтаксис.
```css
.card { padding: 20px; }
.card h2 { font-size: 24px; color: #333; }
.card button { background: #333; color: white; }
.card button:hover { background: #555; }
```

### 2. **Миксин для адаптивности**
Напишите миксин `@mixin mobile`, который добавляет media-query для экранов меньше 600px, и примените его к любому блоку на странице.

### 3. **Разделяй и властвуй**
Изучите концепцию `@import` (или современный `@use`) в SASS. Создайте структуру файлов: `_variables.scss`, `_header.scss` и основной `style.scss`, который их объединяет.

### 4. **Gulp Watch (Исследование)**
Найдите в интернете простой `gulpfile.js` для компиляции SASS. Опишите своими словами, что делает команда `gulp.watch`.
