## |1| **Секреты и dotenv**

Если вы выложите код с паролем от базы данных на GitHub — это катастрофа. Для защиты данных используют **переменные окружения** (Environment Variables).

В Node.js доступ к ним осуществляется через `process.env`. Чтобы было удобно работать с ними локально, используют пакет `dotenv`.

### **Установка**
```bash
npm install dotenv
```

---

## |2| **Файл .env**

Создайте файл `.env` (с точкой в начале) в корне проекта. В нем хранятся пары `КЛЮЧ=ЗНАЧЕНИЕ`.

```text
PORT=3000
DB_PASSWORD=secret123
API_KEY=xyz-789
```

### **Подключение в коде**
```javascript
require('dotenv').config(); // Загружаем переменные из .env

const port = process.env.PORT || 8080;
console.log(`Сервер будет запущен на порту ${port}`);
```

> [!IMPORTANT]
> Всегда добавляйте файл `.env` в свой `.gitignore`, чтобы случайно не опубликовать его.

## |3| **Отправка почты с Nodemailer**

`Nodemailer` — это самый популярный модуль для отправки email в Node.js.

### **Установка**
```bash
npm install nodemailer
```

### **Пример отправки**
```javascript
const nodemailer = require('nodemailer');

// 1. Создаем транспорт (настройки почтового сервера)
const transporter = nodemailer.createTransport({
    service: 'gmail',
    auth: {
        user: process.env.EMAIL_USER,
        pass: process.env.EMAIL_PASS
    }
});

// 2. Настраиваем письмо
const mailOptions = {
    from: 'Goddess <charisma@example.com>',
    to: 'student@example.com',
    subject: 'Привет от Граммы!',
    text: 'Это письмо отправлено автоматически из Node.js'
};

// 3. Отправляем
transporter.sendMail(mailOptions, (error, info) => {
    if (error) return console.log(error);
    console.log('Письмо отправлено: ' + info.response);
});
```

## |4| **Тестирование почты (Ethereal)**

Если у вас нет своего SMTP сервера или вы не хотите использовать личный Gmail, можно создать фейковый аккаунт на [Ethereal Email](https://ethereal.email/). Он перехватывает ваши письма и показывает их в веб-интерфейсе.

## **Итог**

-   **dotenv** — защищает секреты, вынося их за пределы кода.
-   **process.env** — встроенный объект для чтения настроек системы.
-   **Nodemailer** — гибкий инструмент для отправки писем через любые почтовые службы.
-   Никогда не коммитьте `.env` в публичные репозитории!

---

## **Практика**

### 1. **Безопасный старт**
Создайте файл `.env` и добавьте туда переменную `ADMIN_NAME`. Напишите скрипт, который приветствует админа по имени, взятому из окружения. Если переменной нет, должен выводить: "Привет, Гость".

### 2. **Рассыльщик уведомлений**
Реализуйте функцию `sendAlert(message)`, которая отправляет email на ваш адрес с текстом `message`. Используйте `dotenv` для хранения почтовых учетных данных.

### 3. **Динамический Порт**
Настройте сервер так, чтобы он считывал порт из `.env`, а если там пусто — запускался на порту `5000`.

### 4. **HTML-письмо**
Модифицируйте код отправки письма так, чтобы вместо обычного текста (`text`) отправлялось красивое `html` сообщение с жирным текстом и ссылкой.
