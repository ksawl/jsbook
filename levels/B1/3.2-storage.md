# **Хранение данных в браузере**

Современные веб-приложения должны уметь сохранять состояние: настройки темы, содержимое корзины или данные профиля. Для этого в браузере существуют механизмы локального хранения, работающие на стороне клиента.

## |1| **LocalStorage и SessionStorage**

Оба этих хранилища позволяют хранить пары "ключ-значение". Главное отличие между ними — **время жизни данных**.

-   **localStorage**: Данные сохраняются навсегда (или пока пользователь не очистит их вручную). Они остаются даже после перезагрузки страницы или закрытия браузера.
-   **sessionStorage**: Данные существуют только в рамках текущей вкладки. Если закрыть вкладку, данные удаляются.

### **Особенности**
1.  **Лимит**: Около 5 МБ (в зависимости от браузера).
2.  **Тип данных**: Хранить можно **только строки**.
3.  **Безопасность**: Доступно только для того же домена (Same-origin policy).

---

## |2| **Методы работы с хранилищем**

Интерфейс у обоих хранилищ одинаковый:

```javascript
// Сохранение данных
localStorage.setItem('theme', 'dark');

// Получение данных
const theme = localStorage.getItem('theme');
console.log(theme); // 'dark'

// Удаление по ключу
localStorage.removeItem('theme');

// Полная очистка
localStorage.clear();
```

## |3| **Хранение объектов (JSON)**

Так как хранилище принимает только строки, объекты и массивы нужно предварительно превращать в JSON.

```javascript
const user = { name: 'Alice', role: 'admin' };

// Сохраняем
localStorage.setItem('user', JSON.stringify(user));

// Получаем и парсим обратно
const savedUser = JSON.parse(localStorage.getItem('user'));
console.log(savedUser.name); // 'Alice'
```

> [!WARNING]
> Если попытаться сохранить объект напрямую: `localStorage.setItem('key', user)`, браузер превратит его в строку `"[object Object]"`, и данные будут потеряны.

## |4| **Cookies: классический подход**

**Cookies** (Куки) — это небольшие строки данных, которые отправляются на сервер с каждым HTTP-запросом.

-   **Для чего**: Авторизация, отслеживание сессий.
-   **Объем**: Очень маленький (до 4 КБ).
-   **Работа в JS**: Через свойство `document.cookie`.

```javascript
// Установка куки
document.cookie = "user=Ivan; max-age=3600; path=/";

// Чтение (возвращает одну длинную строку всех кук)
console.log(document.cookie); // "user=Ivan; theme=dark"
```
*Для работы с куки в JS обычно используют дополнительные библиотеки, так как ручной парсинг строки `document.cookie` очень неудобен.*

## **Практика**

### 1. **Сохранение темы**
Создайте простую страницу с кнопкой "Переключить тему". При клике меняйте фон страницы и сохраняйте значение темы в `localStorage`. При перезагрузке страницы тема должна восстанавливаться.

### 2. **Призрачная форма**
Создайте текстовое поле `textarea`. Настройте автоматическое сохранение текста в `sessionStorage` при каждом изменении (`input` событие). После закрытия и открытия вкладки текст должен пропасть, а при простой перезагрузке — остаться.

### 3. **Smart JSON Storage**
Напишите функцию-обертку `saveToLocal(key, value)`, которая сама определяет: если передан объект — делает `JSON.stringify`, если строка — сохраняет как есть. И аналогичную функцию `getFromLocal(key)`.

### 4. **Краткосрочные Cookies**
Напишите код, который устанавливает куку `visited=true`, которая удалится сама через 10 секунд. Проверьте её наличие через `console.log(document.cookie)` сразу и спустя 10 секунд.
