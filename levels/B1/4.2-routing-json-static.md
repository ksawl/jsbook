## |1| **Маршрутизация (Routing)**

**Роутинг** — это процесс определения того, какой код должен выполниться в зависимости от URL (пути) и HTTP-метода запроса.

В "чистом" Node.js мы проверяем свойство `req.url` и `req.method`.

```javascript
const http = require('http');

const server = http.createServer((req, res) => {
    if (req.url === '/' && req.method === 'GET') {
        res.end('Главная страница');
    } else if (req.url === '/about') {
        res.end('О проекте');
    } else {
        res.writeHead(404);
        res.end('Страница не найдена');
    }
});
```

## |2| **Отправка JSON**

Для современных приложений (SPA, мобильные приложения) сервер чаще всего возвращает не текст, а данные в формате JSON.

```javascript
const server = http.createServer((req, res) => {
    if (req.url === '/api/user') {
        const user = { id: 1, name: 'Харизма', role: 'Goddess' };
        
        res.writeHead(200, { 'Content-Type': 'application/json' });
        res.end(JSON.stringify(user));
    }
});
```

## |3| **Работа со статическими файлами**

Чтобы отправить файл (например, `index.html`), нужно его прочитать с диска и передать в поток ответа.

```javascript
const fs = require('fs');
const path = require('path');

const server = http.createServer((req, res) => {
    if (req.url === '/') {
        const filePath = path.join(__dirname, 'public', 'index.html');
        
        fs.readFile(filePath, (err, content) => {
            if (err) {
                res.writeHead(500);
                res.end('Ошибка сервера');
                return;
            }
            res.writeHead(200, { 'Content-Type': 'text/html' });
            res.end(content);
        });
    }
});
```

## |4| **Обработка POST запросов**

В отличие от GET, данные в POST запросе приходят по частям (стримами). Нам нужно их "собрать".

```javascript
let body = '';
req.on('data', chunk => {
    body += chunk.toString(); // Собираем части данных
});

req.on('end', () => {
    console.log('Данные получены:', body);
    res.end('Данные приняты!');
});
```

---

## **Итог**

-   **Роутинг** строится на проверке `req.url` и `req.method`.
-   Для отправки **JSON** необходимо установить заголовок `Content-Type: application/json`.
-   **Статика** отдается через модуль `fs`. Помните про правильные MIME-типы (html, css, js).
-   **POST-данные** принимаются как поток событий `'data'` и завершаются событием `'end'`.

## **Практика**

### 1. **Мини-API**
Создайте сервер с роутом `/api/status`, который возвращает JSON объект с полями `status: "ok"` и `uptime: [время работы сервера в секундах]`.

### 2. **Файловый менеджер**
Добавьте роут `/download`, который читает текстовый файл с вашего диска и отправляет его содержимое пользователю в браузер.

### 3. **Сборщик контактов (POST)**
Реализуйте обработку POST запроса на адрес `/contact`. Сервер должен принять JSON с именем и email, распарсить его (`JSON.parse`) и вывести в консоль фразу: "Получен контакт: [Имя] ([email])".

### 4. **Умный роутер**
Напишите функцию `router(req, res)`, которая будет содержать всю логику ветвления, чтобы основной код `http.createServer` выглядел чисто: `http.createServer(router).listen(3000)`.
