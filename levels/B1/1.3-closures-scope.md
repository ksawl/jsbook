# **Замыкания и область видимости**

Замыкания — это "сердце" JavaScript. Они позволяют функциям запоминать окружение, в котором они были созданы, даже после того, как внешняя функция завершила работу.

## |1| **Область видимости (Scope)**

В JavaScript существует три уровня области видимости:
1. **Глобальная**: доступна везде.
2. **Функциональная**: доступна только внутри `function`.
3. **Блочная** (`let`, `const`): доступна внутри `{ ... }`.

При поиске переменной JavaScript идёт от внутренней области к внешней по **цепочке областей видимости** (Scope Chain).

---

## |2| **Что такое замыкание (Closure)?**

**Замыкание** — это функция, которая запоминает своё лексическое окружение (Lexical Environment) в момент создания. Это позволяет функции иметь доступ к переменным внешней функции даже после её завершения.

```javascript
function createCounter() {
  let count = 0; // Переменная "замыкается" внутри
  return function() {
    return ++count;
  };
}

const counter = createCounter();
console.log(counter()); // 1
console.log(counter()); // 2
```

## |3| **Карирование (Currying)**

**Карирование** — это трансформация функции с несколькими аргументами в последовательность функций, каждая из которых принимает только один аргумент: `f(a, b, c)` → `f(a)(b)(c)`.

### **Преимущества**
1. **Частичное применение**: Можно зафиксировать часть аргументов и создать новую специализированную функцию.
2. **Модульность**: Позволяет разбивать сложные функции на простые части.

**Пример (ручное карирование):**
```javascript
const multiply = a => b => a * b;
const double = multiply(2); // Зафиксировали a = 2
console.log(double(10)); // 20
```

**Пример (автоматический утилита-карри):**
```javascript
function curry(fn) {
  return function curried(...args) {
    if (args.length >= fn.length) {
      return fn.apply(this, args);
    }
    return (...moreArgs) => curried.apply(this, args.concat(moreArgs));
  };
}
```

## |4| **Практическое применение**

### 1. **Создание приватных переменных**
Замыкания позволяют имитировать «приватность», скрывая данные от прямого доступа.
```javascript
function createPerson(name) {
  let age = 0; // Скрыто
  return {
    getName: () => name,
    getAge: () => age,
    grow: () => age++
  };
}
```

### 2. **Мемоизация (Кэширование)**
Ускорение выполнения функций за счет сохранения результатов предыдущих вызовов.
```javascript
function memoize(fn) {
  const cache = {};
  return function(arg) {
    if (cache[arg] !== undefined) return cache[arg];
    const result = fn(arg);
    cache[arg] = result;
    return result;
  };
}
```

---

## **Итог**

- **Замыкание** — это «память» функции о том, где она родилась.
- **Карирование** делает код более гибким и функциональным.
- **Мемоизация** — мощный инструмент оптимизации производительности.

## **Практика**

### 1. **Счетчик с шагом**
Напишите функцию `createIncrementer(step)`, которая возвращает функцию, увеличивающую число на переданный `step`.

### 2. **Таймер обратного отсчета**
Напишите функцию `countdown(n)`, которая при каждом вызове возвращает `n-1`, а когда доходит до 0 — выводит "Время вышло!".

### 3. **Мемоизация факториала**
Используя функцию `memoize` из примера выше, оберните в неё рекурсивную функцию факториала и замерьте время (через `console.time`) для первого и второго вызова.
