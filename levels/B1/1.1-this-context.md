# **Контекст вызова: this**

Контекст `this` в JavaScript — это одна из самых важных тем. Значение `this` зависит от того, **как** и **где** функция была вызвана, а не от того, где она была объявлена.

## |1| **Что такое this?**

`this` — это ключевое слово, которое ссылается на объект, в контексте которого выполняется текущий код. Оно позволяет методам объекта обращаться к другим свойствам этого же объекта.

Значение `this` определяется в момент вызова функции и может меняться в зависимости от того, **как** вызывается функция, а не от того, где она объявлена.

---

## |2| **this в различных контекстах**

### 1. **Глобальный контекст**
Вне функций (в глобальной области видимости) `this` ссылается на глобальный объект:
- В браузере — это `window`.
- В Node.js — это `global`.

```javascript
console.log(this); // В браузере: window, в Node.js: global
```

### 2. **Обычные функции (Именованные и Анонимные)**
Значение `this` в обычных функциях зависит от режима и способа вызова:
- **Без контекста**: В нестрогом режиме — глобальный объект. В `'use strict'` — `undefined`.
- **Методы объектов**: `this` ссылается на объект перед точкой: `obj.method()`.

---

#### 💡 **Нюанс: Отладка анонимных функций**
Хотя поведение `this` в анонимных и именованных функциях идентично, есть важное различие в **отладке**:
- **Именованные функции** легче идентифицировать в стеке вызовов (Stack Trace), так как они имеют имя.
- **Анонимные функции** отображаются как `(anonymous)`, что затрудняет поиск ошибки в сложных цепочках вызовов.

---

### 3. **Стрелочные функции**
Стрелочные функции (**arrow functions**) **не имеют своего собственного this**. Вместо этого они захватывают значение `this` из окружающего лексического контекста (там, где они были определены).

**Пример (потеря контекста в setTimeout):**
```javascript
const obj = {
  name: 'Alice',
  greet: function() {
    // ОБЫЧНАЯ функция внутри setTimeout потеряет контекст
    setTimeout(function() {
      console.log(`Привет, я ${this.name}`); // this === window/undefined
    }, 100);

    // СТРЕЛОЧНАЯ функция захватит this из greet (т.е. obj)
    setTimeout(() => {
      console.log(`Привет, я ${this.name}`); // this === obj
    }, 200);
  }
};
```

---

### 4. **Контекст в IIFE**
Поскольку IIFE (Immediately Invoked Function Expression) вызывается сразу и без явного контекста, `this` внутри неё обычно ссылается на глобальный объект (или `undefined` в строгом режиме). Она **не наследует** `this` из окружающего метода автоматически.

---

## |3| **Управление контекстом: call, apply, bind**

Эти методы позволяют явно указать значение `this` при вызове функции.

- **`call(context, arg1, arg2...)`** — вызывает функцию немедленно с указанным контекстом и аргументами через запятую.
- **`apply(context, [args])`** — то же самое, но аргументы передаются массивом.
- **`bind(context)`** — возвращает новую функцию, навсегда привязанную к контексту.

> [!CAUTION]
> Стрелочные функции **не могут** изменить значение `this` с помощью `call`, `apply` или `bind`. Их `this` всегда остается неизменным.

---

## |4| **Scope vs Context**

Эти понятия часто путают, но они фундаментально разные:

| Параметр | **Область видимости (Scope)** | **Контекст (Context)** |
| :--- | :--- | :--- |
| **Зависимость** | Где функция *объявлена* | Как функция *вызвана* |
| **Ключевое слово** | `let`, `const`, `var` | `this` |
| **Фокус** | Доступ к переменным | Указывает на объект-владелец |

---

## **Итог**

- `this` определяется в момент вызова («кто вызвал?»).
- Стрелочные функции используют лексический `this` («где я родился?»).
- `bind` — лучший способ зафиксировать контекст для колбэков.
- Помните про различие между тем, *что* функция видит (Scope), и тем, *кем* она себя считает (Context).

## **Практика**

### 1. **Контекст в IIFE**
Что выведет код и почему?
```javascript
const user = {
  name: 'Грамма',
  run() {
    (function() {
      console.log(this.name);
    })();
  }
};
user.run();
```
