## |1| **Особенности асинхронности в JS**

JavaScript — **однопоточный** язык. Это значит, что он может выполнять только одну задачу в один момент времени. Асинхронность — это способ не "замораживать" программу, пока мы ждем ответа от сервера или срабатывания таймера.

### **Инструменты асинхронности**
1. **Коллбэки**: Базовый (старый) способ.
2. **Промисы**: Современный стандарт управления успехом/ошибкой.
3. **Async/Await**: Синтаксический сахар для удобного чтения.

---

## |2| **Коллбэки (Callbacks)**

**Коллбэк** — это функция, переданная как аргумент, которая будет вызвана позже.

### 1. **Callback Hell (Ад колбэков)**
Когда задачи зависят друг от друга, код превращается в неуправляемую "пирамиду" (Pyramid of Doom), которую невозможно читать и тестировать.

```javascript
doStep1(() => {
    doStep2(() => {
        doStep3(() => {
            console.log('Готово!');
        });
    });
});
```

## |3| **Таймеры: setTimeout и setInterval**

JavaScript позволяет планировать выполнение кода во времени.

### 1. **setTimeout** (Один раз)
```javascript
const timerId = setTimeout((msg) => {
    console.log(msg);
}, 2000, 'Прошло 2 секунды');
```

### 2. **setInterval** (Повторно)
```javascript
const intervalId = setInterval(() => console.log('Tik'), 1000);
setTimeout(() => clearInterval(intervalId), 5000); // Остановим через 5 сек
```

> [!IMPORTANT]
> Таймеры работают через **Event Loop**. Если основной поток занят тяжелыми вычислениями, таймер сработает позже, чем планировалось.

## |4| **Рекурсия: логика самовызова**

**Рекурсия** — это когда функция вызывает саму себя. У неё всегда должны быть:
1.  **Базовый случай** — условие выхода.
2.  **Рекурсивный шаг** — движение к выходу.

### **Сравнение: Рекурсия vs Итерация**

| Критерий | **Рекурсия** | **Итерация (Циклы)** |
| :--- | :--- | :--- |
| **Читаемость** | Выше для сложных структур (деревья) | Выше для простых списков |
| **Память** | Расходует стек вызовов | Экономнее (один цикл) |
| **Риск** | Stack Overflow при большой глубине | Бесконечный цикл (зависание) |

### **Классический пример: Обход дерева**
Рекурсия незаменима для работы со вложенными структурами (DOM, JSON-файлы).

```javascript
const tree = {
    value: 1,
    children: [
        { value: 2, children: [{ value: 4, children: [] }] },
        { value: 3, children: [] }
    ],
};

function traverse(node) {
    console.log(node.value);
    node.children.forEach(child => traverse(child));
}
traverse(tree); // 1, 2, 4, 3
```

## |5| **Хвостовая оптимизация (TCO)**

Если рекурсивный вызов — самое последнее действие в функции, некоторые движки (Safari) "схлопывают" стек, предотвращая падение.

```javascript
function sum(n, acc = 0) {
    if (n === 0) return acc;
    return sum(n - 1, acc + n); // TCO-friendly
}
```

## **Итог**

- **Коллбэки** — "сделай это, когда освободишься".
- **Таймеры** — планирование задач в Event Loop.
- **Рекурсия** — элегантный способ работы с глубиной, но берегите стек!
- **Базовый случай** — залог безопасности рекурсии.
