# **Глубокое копирование**

В JavaScript понимание того, как копируются объекты, — это разница между стабильным приложением и кодом, полным "магических" багов. Поскольку объекты передаются по ссылке, простое присваивание не создает новый объект, а лишь дает еще один "пульт управления" тем же самым участком памяти. В этом уроке мы научимся делать настоящие копии.

## |1| **Поверхностное копирование (Shallow Copy)**

Поверхностное копирование создает новый объект, но копирует в него только значения свойств первого уровня. Если внутри объекта есть другие объекты или массивы, они все еще будут ссылаться на ту же самую память.

### 1. **Spread-оператор (`...`)**
Самый современный и лаконичный способ.

```javascript
const user = { name: 'Alice', city: 'Paris' };
const copy = { ...user };

copy.name = 'Bob';
console.log(user.name); // Alice (не изменилось)
```

### 2. **Метод `Object.assign()`**
Более старый способ, часто встречается в легаси-коде.

```javascript
const user = { name: 'Alice' };
const copy = Object.assign({}, user);
```

### **Главная проблема: Вложенность**
Если в объекте есть вложенные структуры, "поверхностный" метод скопирует только ссылку на них.

```javascript
const user = { 
    name: 'Alice', 
    address: { city: 'Paris' } 
};
const copy = { ...user };

copy.address.city = 'London';
console.log(user.address.city); // London! (Оригинал пострадал)
```

## |2| **Глубокое копирование (Deep Copy)**

Для создания полной, независимой копии всех уровней вложенности используются другие методы.

### 1. **Современный стандарт: `structuredClone()`**
Это встроенный метод JavaScript (доступен в Node.js 17+ и современных браузерах). Он умеет копировать даже сложные структуры, цикличные ссылки и различные типы данных.

```javascript
const user = { 
    name: 'Alice', 
    address: { city: 'Paris' } 
};
const deepCopy = structuredClone(user);

deepCopy.address.city = 'London';
console.log(user.address.city); // Paris (Оригинал в безопасности)
```

### 2. **JSON-хак: `JSON.parse(JSON.stringify())`**
Старый "народный" метод. Объект превращается в строку, а затем обратно в новый объект.

**Минусы:**
-   **Теряет данные**: функции, `undefined`, символы, `Date` (превращает в строку) исчезают или искажаются.
-   **Медленнее** на очень больших объектах.

```javascript
const user = { name: 'Alice', date: new Date() };
const fastCopy = JSON.parse(JSON.stringify(user));

console.log(typeof fastCopy.date); // string (Потерян тип Date!)
```

### 3. **Рекурсивный обход**
До появления `structuredClone` программисты писали свои функции, которые циклом обходят объект и, встречая другой объект внутри, вызывают сами себя. Это самый мощный, но сложный способ.

---

## |3| **Когда что использовать?**

| Метод | Тип копии | Вложенность | Когда использовать |
| :--- | :--- | :--- | :--- |
| `{ ...obj }` | Поверхностная | ❌ Нет | Когда объект простой (без вложенности) |
| `Object.assign` | Поверхностная | ❌ Нет | В старом коде или при слиянии объектов |
| `structuredClone` | Глубокая | ✅ Да | **Рекомендуемый стандарт** для глубоких копий |
| `JSON.parse` | Глубокая | ✅ Да | Если нужно поддержать очень старую среду и объект простой (только строки/числа) |

---

## **Итог**

-   **Поверхностное копирование** (`...`) работает быстро, но безопасно только для плоских объектов.
-   Если в объекте есть "объекты в объектах" — используйте **глубокое копирование**.
-   **`structuredClone()`** — ваш основной инструмент для глубоких копий в 2024 году.

## **Практика**

### 1. **Ошибка в системе заказов**
Исправьте код. При изменении адреса в `newOrder` адрес в `originalOrder` тоже меняется. Сделайте копию независимой.
```javascript
const originalOrder = {
    id: 1,
    items: ['Apple', 'Banana'],
    shipping: { city: 'Moscow' }
};

const newOrder = { ...originalOrder };
newOrder.shipping.city = 'Spb';
```

### 2. **Тест JSON-хака**
Создайте объект `weirdObj`, у которого одно свойство — это функция, а другое — `undefined`. Сделайте копию через `JSON.parse(JSON.stringify())`. Выведите копию в консоль. Что пропало? Почему?

### 3. **Сравнение сущностей**
Напишите функцию `isPureCopy(original, copy)`. Она должна возвращать `true`, если `original.data !== copy.data` (т.е. вложенные объекты имеют разные ссылки), и `false`, если ссылки совпадают. Проверьте её на spread-копии и на `structuredClone`.

### 4. **Клонирование массива объектов**
У вас есть список пользователей: `const users = [{id: 1, info: {name: 'A'}}, {id: 2, info: {name: 'B'}}]`. 
Сделайте глубокую копию этого массива так, чтобы изменение `users[0].info.name` в копии не затронуло оригинал. Используйте `structuredClone`.

### 5. **Многоуровневая структура**
Создайте объект "Дерево", где есть уровни `branch -> subBranch -> leaf`. Попробуйте скопировать его тремя способами: spread, `JSON` и `structuredClone`. В каждом случае измените "цвет листа" (`leaf.color`) и проверьте, как это повлияло на исходное дерево.
