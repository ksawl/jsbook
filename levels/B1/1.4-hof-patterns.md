## |1| **Что такое HOF?**

**Функция высшего порядка (Higher-Order Function)** — это функция, которая воспринимает другие функции как данные. Она либо:
1. **Принимает функцию** как аргумент (например, `map`, `filter`).
2. **Возвращает функцию** как результат (например, фабрики функций или декораторы).

### **Зачем это нужно?**
- **Абстракция**: Мы передаём "действие", а не просто данные. Это позволяет скрыть детали реализации циклов или проверок.
- **Композиция**: Возможность собирать сложные операции из простых функций-кирпичиков.
- **Декларативность**: Код описывает *что* мы хотим получить, а не *как* пошагово это делать.

---

## |2| **Композиция функций**

Один из мощнейших паттернов HOF — объединение нескольких простых функций в одну сложную.

```javascript
const compose = (f, g) => (x) => f(g(x));

const addOne = x => x + 1;
const double = x => x * 2;

const addAndDouble = compose(double, addOne);
console.log(addAndDouble(5)); // (5 + 1) * 2 = 12
```

## |3| **Паттерн Debounce**

**Debounce (Устранение дребезга)** — техника, которая заставляет функцию ждать определенное время после последнего вызова, прежде чем выполниться.

**Зачем это нужно:** Оптимизация поиска при вводе текста. Мы не хотим отправлять запрос на сервер после каждой нажатой буквы.

```javascript
function debounce(fn, delay) {
  let timeout;
  return (...args) => {
    clearTimeout(timeout);
    timeout = setTimeout(() => fn.apply(this, args), delay);
  };
}
```

---

## |4| **Паттерн Throttle**

**Throttle (Троттлинг)** — ограничение частоты выполнения функции. Она будет выполняться не чаще, чем один раз в указанный период.

**Зачем это нужно:** Обработка скролла или изменения размера окна.

```javascript
function throttle(fn, limit) {
  let inThrottle;
  return (...args) => {
    if (!inThrottle) {
      fn.apply(this, args);
      inThrottle = true;
      setTimeout(() => (inThrottle = false), limit);
    }
  };
}
```

---

## |5| **Мемоизация**

Кэширование результатов функции на основе её аргументов. Повторный вызов с теми же данными мгновенно возвращает результат из памяти.

```javascript
function memoize(fn) {
  const cache = new Map();
  return (...args) => {
    const key = JSON.stringify(args);
    if (cache.has(key)) return cache.get(key);
    const result = fn(...args);
    cache.set(key, result);
    return result;
  };
}
```

---

## **Итог**

- **Debounce** — "жди, пока я закончу печатать".
- **Throttle** — "делай это не чаще, чем раз в секунду".
- **Композиция** — "собирай сложное из простого".
- **HOF** — фундамент для создания таких инструментов оптимизации.

## **Практика**

### 1. **Умный поиск**
Реализуйте Debounce для функции вывода в консоль, чтобы сообщение выводилось только через 500мс после того, как пользователь "перестал нажимать клавиши".

### 2. **Экономичный скролл**
Напишите Throttle, который выводит текущую координату скролла страницы не чаще раза в 200мс.

### 3. **Тройная композиция**
Создайте функцию `compose`, которая принимает три функции и возвращает их композицию. Проверьте на функциях: возведение в квадрат, прибавление 10 и преобразование в строку с префиксом.
