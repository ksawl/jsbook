# **Продвинутая работа с событиями**

События в браузере — это не просто "клик по кнопке". Это сложный механизм взаимодействия элементов, понимание которого превращает обычного верстальщика в настоящего разработчика. В этом уроке мы разберем, как события движутся по DOM-дереву и как ими управлять.

## |1| **Объект события (event)**

Когда происходит событие, браузер создает объект события, содержащий детали о нем, и передает его в качестве аргумента в обработчик.

```javascript
document.querySelector('button').addEventListener('click', (event) => {
    console.log(event.type);    // 'click'
    console.log(event.target);  // Элемент, на котором произошло событие
    console.log(event.clientX); // Координата X курсора
});
```

## |2| **Всплытие и погружение (Bubbling & Capturing)**

Представьте, что внутри `<div>` находится `<p>`. Если кликнуть по `<p>`, событие сначала "погружается" сверху вниз, а затем "всплывает" снизу вверх через всех родителей.

1.  **Погружение (Capturing)**: Событие идет от `window` к целевому элементу.
2.  **Цель (Target)**: Событие достигло элемента.
3.  **Всплытие (Bubbling)**: Событие поднимается от элемента обратно к `window`.

**По умолчанию** почти все обработчики работают на стадии **всплытия**.

## |3| **Делегирование событий**

Это мощный паттерн, позволяющий не вешать обработчик на каждую кнопку, а повесить один общий на их родителя.

### **Зачем это нужно?**
-   Экономия памяти.
-   Удобство: если вы добавите новую кнопку динамически, она сразу будет работать.

### **Пример**
```javascript
const menu = document.querySelector('#menu');

menu.addEventListener('click', (event) => {
    if (event.target.tagName === 'BUTTON') {
        console.log('Нажата кнопка:', event.target.textContent);
    }
});
```

## |4| **Управление поведением (preventDefault & stopPropagation)**

### **1. event.preventDefault()**
Отменяет стандартное действие браузера. Например, переход по ссылке или отправку формы.
```javascript
link.addEventListener('click', (e) => {
    e.preventDefault(); // Ссылка не сработает
    console.log('Мы никуда не летим!');
});
```

### **2. event.stopPropagation()**
Останавливает дальнейшее "всплытие" события. Родители не узнают, что на ребенке что-то произошло.
```javascript
button.addEventListener('click', (e) => {
    e.stopPropagation(); // Событие умрет здесь
});
```

---

## **Итог**

-   **Всплытие** — событие идет от глубокого элемента вверх к родителю.
-   **Делегирование** — используем всплытие, чтобы обрабатывать события на родителе.
-   **preventDefault** — запрещаем браузеру делать то, что он привык (переход, отправка).
-   **stopPropagation** — запрещаем событию лететь выше к родителям.

## **Практика**

### 1. **Детектор Кнопок**
Создайте список `<ul>` с 10 элементами `<li>`, внутри каждого из которых есть кнопка "Удалить". Используя **делегирование**, сделайте так, чтобы при клике на любую кнопку консоль выводила текст: "Удален элемент номер [индекс]".

### 2. **Запретная Зона**
Создайте ссылку на любой сайт. Сделайте так, чтобы при клике на неё браузер не переходил по адресу, а вместо этого показывал скрытый текстовый блок на странице.

### 3. **Тройная Матрешка**
Создайте три вложенных `<div>` разного цвета. Повесьте на каждый обработчик клика. Сделайте так, чтобы при клике на самый внутренний блок срабатывал ТОЛЬКО его обработчик, а внешние "молчали".

### 4. **Интерактивная Галерея**
Создайте блок-контейнер с несколькими миниатюрами картинок. Используя делегирование, сделайте так, чтобы при клике на миниатюру её `src` заменял `src` большой картинки, расположенной рядом.
