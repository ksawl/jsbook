На этом этапе мы изучим, как контролировать поведение свойств, использовать мощь функций-конструкторов и почему некоторые старые методы лучше оставить в прошлом.

## |1| **Геттеры и Сеттеры**

**Геттеры** (`get`) и **сеттеры** (`set`) — это специальные методы доступа к свойствам, которые выглядят снаружи как обычные поля, но внутри скрывают вызов функции.

Они позволяют:
-   **Вычислять** значения на лету.
-   **Валидировать** (проверять) данные перед записью.
-   **Скрывать** внутреннее состояние объекта (инкапсуляция).

### 1. **Синтаксис**

Геттеры и сеттеры объявляются прямо внутри литерала объекта или класса.

```javascript
const user = {
    firstName: 'Alice',
    lastName: 'Smith',

    // Геттер: вызывается при чтении fullName
    get fullName() {
        return `${this.firstName} ${this.lastName}`;
    },

    // Сеттер: вызывается при попытке записи в fullName
    set fullName(value) {
        const parts = value.split(' ');
        if (parts.length < 2) {
            console.error('Ошибка: неверный формат имени');
            return;
        }
        [this.firstName, this.lastName] = parts;
    }
};

console.log(user.fullName); // Alice Smith (сработал get)
user.fullName = 'Bob Johnson'; // (сработал set)
console.log(user.firstName); // Bob
```

### 2. **Валидация данных**

Сеттеры идеально подходят для защиты логики. Например, мы не хотим, чтобы возраст был отрицательным.

```javascript
class User {
    constructor(age) {
        this._age = age; // Символ подчеркивания — соглашение о "приватном" свойстве
    }

    get age() {
        return this._age;
    }

    set age(value) {
        if (value < 0) {
            console.error('Возраст не может быть меньше нуля!');
            return;
        }
        this._age = value;
    }
}

const user = new User(25);
user.age = -5; // Ошибка в консоли, значение не изменится
```

## |2| **Дескрипторы свойств**

До этого момента мы просто добавляли свойства: `obj.prop = value`. Но JavaScript позволяет настроить каждое свойство гораздо тоньше с помощью **дескрипторов**.

Метод `Object.defineProperty(obj, propertyName, descriptor)` позволяет гибко настроить права доступа к свойству.

### 1. **Флаги свойств**

Каждое свойство объекта имеет четыре специальных атрибута (флага):
-   `value` — текущее значение.
-   `writable` — если `true`, значение можно изменять, иначе оно только для чтения.
-   `enumerable` — если `true`, свойство видно в циклах `for..in` и `Object.keys()`.
-   `configurable` — если `true`, свойство можно удалить, а эти флаги — изменить.

### 2. **Пример использования**

```javascript
const user = {};

Object.defineProperty(user, 'id', {
    value: 12345,
    writable: false,      // Нельзя перезаписать
    configurable: false,   // Нельзя удалить
    enumerable: true      // Видно в циклах
});

user.id = 999; // Значение не изменится (в строгом режиме будет ошибка)
delete user.id; // Не удалится
console.log(user.id); // 12345
```

### 3. **Групповое определение**

Метод `Object.defineProperties` позволяет настроить сразу несколько полей:

```javascript
Object.defineProperties(user, {
    name: { value: 'Alice', writable: true },
    role: { value: 'admin', writable: false }
});
```

## |3| **Функции-конструкторы и Фабрики**

Когда нам нужно создать много однотипных объектов (например, пользователей или товаров), мы используем специальные паттерны.

### 1. **Функции-конструкторы**

Это обычные функции, которые вызываются с ключевым словом `new`. По соглашению их имена начинаются с Большой буквы.

-   Внутри создается новый пустой объект и присваивается `this`.
-   Функция наполняет `this` свойствами.
-   Объект возвращается автоматически.

```javascript
function User(name, role) {
    this.name = name;
    this.role = role;
}

// ПРАВИЛЬНО: Методы добавляем в прототип
User.prototype.greet = function() {
    console.log(`Привет, я ${this.name}`);
};

const admin = new User('Ivan', 'admin');
```

> [!IMPORTANT]
> **Оптимизация памяти**: Если объявлять методы внутри конструктора через `this.method = ...`, то при создании 1000 объектов будет создано 1000 копий функции. Если добавить метод в `prototype`, он будет существовать в единственном экземпляре в памяти, и все объекты будут ссылаться на него.

### 2. **Объекты-словари (`Object.create(null)`)**

Обычный объект `{}` наследует методы от `Object.prototype` (например, `toString`). Иногда нам нужен "чистый" словарь без лишнего багажа.

```javascript
const dict = Object.create(null);
dict.key = 'value';

console.log(dict.toString); // undefined
console.log(dict instanceof Object); // false
```
Это полезно для хранения данных, где ключи могут совпадать с именами встроенных методов (например, ключ `"hasOwnProperty"`).

## |4| **Сравнение способов создания объектов**

| Способ | Прототип | Когда использовать |
| :--- | :--- | :--- |
| **Литерал `{}`** | `Object.prototype` | Простые структуры данных. |
| **`new Object()`** | `Object.prototype` | Редко, аналог литерала. |
| **`Object.create(proto)`** | Переданный `proto` | Наследование без классов. |
| **`Object.create(null)`** | Отсутствует | "Чистые" словари для данных. |
| **Конструктор (`new`)** | `Constructor.prototype` | Множество однотипных объектов (B1). |
| **Класс (`class`)** | `Class.prototype` | Современный стандарт ООП (B2). |

## |5| **Оператор with (Legacy)**

Оператор **`with`** — это устаревший способ сокращения пути к свойствам объекта.

**Синтаксис:**
```javascript
const user = { name: 'Alice', age: 25 };

with (user) {
    console.log(name); // Вместо user.name
    console.log(age);  // Вместо user.age
}
```

### **Почему НЕ стоит его использовать?**

1.  **Неоднозначность**: Если в `with` используется имя переменной, которой нет в объекте, JS ищет её во внешней области. Это рождает трудноуловимые баги.
2.  **Строгий режим**: В `"use strict"` оператор `with` **запрещен** и вызывает ошибку.
3.  **Производительность**: Движки JavaScript не могут эффективно оптимизировать код внутри `with`.

**Альтернатива**: Используйте **деструктуризацию**, которую мы изучили ранее.
```javascript
const { name, age } = user;
console.log(name, age);
```

## **Итог**

-   **Геттеры/Сеттеры** превращают простые свойства в "умные", добавляя валидацию и вычисления.
-   **Дескрипторы** позволяют сделать свойства константными (`writable: false`) или скрытыми (`enumerable: false`).
-   **Конструкторы** (`new`) и **Фабрики** — основные инструменты для массового производства объектов.
-   **with** — легаси-инструмент. Знать о нем нужно, использовать — нет.

## **Практика**

### 1. **Умный кошелек**
Создайте объект `wallet` со свойством `_balance`. Реализуйте:
-   Геттер `balance`, возвращающий строку вида `"На счету: X руб."`.
-   Сеттер `balance`, который прибавляет значение к текущему балансу, но выдает ошибку, если число отрицательное.

### 2. **Защищенная Константа**
Создайте пустой объект `config`. С помощью `Object.defineProperty` добавьте в него свойство `API_URL` со значением `"https://api.myapp.com"`. Сделайте его неизменяемым и защищенным от удаления. Проверьте результат, попытавшись перезаписать его.

### 3. **Фабрика Транспорта**
Напишите фабричную функцию `makeCar(model, speed)`, которая возвращает объект машины. Добавьте в объект метод `drive()`, который выводит в консоль: `"Машина [model] едет со скоростью [speed] км/ч"`.

### 4. **Валидация через Сеттер**
Создайте класс `Product`. При создании через конструктор задается `name` и `price`. Используйте сеттер для `price`, чтобы цена не могла быть ниже 10. Если в сеттер передается цена меньше 10 — установите 10 по умолчанию и выведите предупреждение.

### 5. **Почему with — зло?**
Дан код:
```javascript
const settings = { theme: 'dark' };
let theme = 'light';

with (settings) {
    theme = 'blue';
}

console.log(settings.theme); // Что выведет?
console.log(theme);          // А здесь?
```
Проанализируйте и объясните результат, основываясь на теории об операторе `with`.
