## |1| **Сетевая модель и протоколы**

Прежде чем писать код, нужно понять, как данные летают по сети.

-   **IP (Internet Protocol)** — адрес компьютера в сети (как почтовый адрес дома).
-   **TCP (Transmission Control Protocol)** — протокол, который гарантирует, что данные дойдут целиком и в правильном порядке. Это "надежное соединение".
-   **HTTP (HyperText Transfer Protocol)** — протокол "надстрой" над TCP. Основной язык общения веб-браузеров и серверов (запрос-ответ).

Представьте TCP как телефонный звонок, а HTTP — как сам разговор по этому телефону.

## |2| **HTTP Сервер (http.createServer)**

Модуль `http` — это встроенный инструмент Node.js для работы с вебом.

### **Минимальный сервер**
```javascript
const http = require('http');

const server = http.createServer((req, res) => {
    // req (request) — объект запроса от клиента
    // res (response) — объект нашего ответа клиенту

    res.writeHead(200, { 'Content-Type': 'text/plain; charset=utf-8' });
    res.end('Привет, Харизма! Сервер работает.');
});

server.listen(3000, () => {
    console.log('Сервер запущен на http://localhost:3000');
});
```

### **Основные методы Response**
-   `res.writeHead(status, headers)` — отправка заголовков и статус-кода.
-   `res.write(data)` — отправка части данных (можно вызывать несколько раз).
-   `res.end(data)` — завершение ответа и отправка финального куска данных.

## |3| **TCP Сервер и Клиент (модуль net)**

Если HTTP — это разговор, то TCP — это сама трубка. Модуль `net` позволяет работать на более низком уровне.

### **TCP Сервер**
```javascript
const net = require('net');

const server = net.createServer((socket) => {
    socket.write('Вы подключились к TCP серверу\n');
    
    socket.on('data', (data) => {
        console.log(`Получено от клиента: ${data}`);
        socket.write(`Вы сказали: ${data}`);
    });
});

server.listen(8080, '127.0.0.1');
```

## |4| **DNS: Поиск адресов**

Компьютеры общаются по IP-адресам (например, `142.250.186.46`), но люди используют доменные имена (`google.com`). Модуль `dns` помогает переводить одно в другое.

```javascript
const dns = require('dns');

dns.lookup('google.com', (err, address, family) => {
    console.log('IP-адрес:', address);
});

dns.reverse('8.8.8.8', (err, hostnames) => {
    console.log('Хост для Google DNS:', hostnames);
});
```

## **Итог**

-   **HTTP** — протокол для веб-сайтов (Запрос -> Ответ).
-   **TCP** — низкоуровневый протокол для передачи потока данных.
-   **DNS** — телефонная книга интернета.
-   В Node.js для этого используются встроенные модули `http`, `net` и `dns`.

## **Практика**

### 1. **Динамический HTTP Ответ**
Создайте HTTP сервер, который при каждом запросе выводит в консоль метод запроса (`req.method`) и текущее время. В браузере сервер должен отвечать: "Ваш метод: GET, время: [текущее время]".

### 2. **Эхо-сервер TCP**
Напишите TCP сервер, который принимает любые данные от клиента и отправляет их обратно в верхнем регистре (UPPERCASE).

### 3. **DNS Чекер**
Создайте скрипт, который принимает имя домена из переменной и выводит его IP-адрес, а также семейство протокола (IPv4 или IPv6).

### 4. **Умный порт**
Измените HTTP сервер так, чтобы он пытался запуститься на порту 3000, но если он занят (ошибка `'EADDRINUSE'`), выводил в консоль: "Порт занят, попробуйте другой".
*Подсказка: используйте `server.on('error', ...)`.*
