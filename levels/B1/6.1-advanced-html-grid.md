## |1| **Сетки CSS Grid**

Если Flexbox — это про "одномерные" ряды, то **CSS Grid** — это про "двумерные" сетки (строки и колонки одновременно).

### **Основные свойства**
```css
.container {
  display: grid;
  grid-template-columns: 1fr 2fr 1fr; /* Три колонки */
  grid-template-rows: auto 100px;     /* Две строки */
  gap: 20px;                          /* Отступы */
}

.item {
  grid-column: 1 / 3; /* Занимает колонки с 1 по 3 */
}
```

### **repeat и minmax**
```css
grid-template-columns: repeat(auto-fit, minmax(200px, 1fr));
```
Эта магия позволяет создавать адаптивную сетку без использования медиазапросов! Колонки будут подстраиваться под ширину экрана.

## |2| **Адаптивные изображения (<picture>, <source>)**

Загружать огромную картинку на телефон — плохая практика. Тег `<picture>` позволяет браузеру выбирать лучший вариант в зависимости от экрана.

```html
<picture>
  <source srcset="image-large.webp" media="(min-width: 1200px)">
  <source srcset="image-medium.webp" media="(min-width: 800px)">
  <img src="image-small.jpg" alt="Описание">
</picture>
```

## |3| **Группировка контента (<figure>, <details>)**

Семантика — это важно для SEO и доступности.

-   **`<figure>`** — для вставки изображений, схем или кода с подписью `<figcaption>`.
-   **`<details>`** — для создания сворачивающихся блоков (аккордеонов) без JavaScript. Внутри обязательно должен быть заголовок `<summary>`.

```html
<details>
  <summary>Нажми меня, чтобы узнать секрет</summary>
  <p>Секрет в том, что ты молодец!</p>
</details>
```

---

## |4| **Шаблоны (<template>)**

Тег `<template>` содержит HTML-код, который **не отображается** на странице при загрузке. Его можно "оживить" через JavaScript. Это база для создания динамических интерфейсов.

```html
<template id="user-card">
  <div class="card">
    <h3 class="name"></h3>
    <button>Профиль</button>
  </div>
</template>
```

---

## **Итог**

-   **Grid** — лучший способ создания сложных макетов страниц.
-   **`<picture>`** и **`<source>`** — стандарт для производительного адаптива.
-   **Семантика** (`figure`, `details`) делает сайт понятным для роботов и людей.
-   **`<template>`** — удобный контейнер для динамических элементов.

## **Практика**

### 1. **Адаптивная сетка карточек**
Создайте страницу с 6 карточками товаров. Используйте CSS Grid с `repeat(auto-fit, minmax(250px, 1fr))`. Проверьте, как сетка перестраивается при изменении размера окна браузера.

### 2. **Стилизованный аккордеон**
Сделайте FAQ (Часто задаваемые вопросы), используя только теги `<details>` и `<summary>`. Стилизуйте их через CSS так, чтобы при наведении курсора менялся фон заголовка.

### 3. **Картинка под экран**
Добавьте на страницу тег `<picture>`, который показывает вертикальное фото для телефона и горизонтальное для компьютера. Используйте свои фото или картинки-заглушки.

### 4. **Шаблонный список**
Создайте `<template>` для элемента списка дел. Напишите небольшой скрипт, который по нажатию кнопки копирует содержимое шаблона, вставляет туда текст из `prompt` и добавляет новый `<li>` в имеющийся список `<ul>`.
