# **7.1. Введение в Git**

## Что такое система контроля версий?

Система контроля версий (VCS - Version Control System) — это инструмент, который записывает изменения в файлах с течением времени, позволяя восстанавливать определенные версии позже.

### Зачем нужен Git?

- **Отслеживание изменений** — видеть, что изменилось в коде
- **Возможность отката** — вернуться к предыдущей версии
- **Совместная работа** — несколько человек могут работать над одним проектом
- **Резервное копирование** — код хранится в удаленных репозиториях
- **Ветвление** — работа над разными функциями параллельно

## Установка Git

### Windows

1. Скачайте с [git-scm.com](https://git-scm.com/)
2. Запустите установщик
3. Следуйте инструкциям (настройки по умолчанию подходят)

### macOS

```bash
# Через Homebrew
brew install git

# Или через Xcode Command Line Tools
xcode-select --install
```

### Linux (Ubuntu/Debian)

```bash
sudo apt update
sudo apt install git
```

### Проверка установки

```bash
git --version
```

## Первоначальная настройка

### Конфигурация пользователя

```bash
# Установка имени пользователя
git config --global user.name "Ваше Имя"

# Установка email
git config --global user.email "ваш.email@example.com"

# Проверка настроек
git config --list
```

### Настройка редактора

```bash
# Установка VS Code как редактор по умолчанию
git config --global core.editor "code --wait"

# Или другие редакторы
git config --global core.editor "nano"
git config --global core.editor "vim"
```

### Другие полезные настройки

```bash
# Автоматическое исправление ошибок в командах
git config --global help.autocorrect 1

# Цветной вывод
git config --global color.ui auto

# Настройка поведения при pull
git config --global pull.rebase false
```

## Создание репозитория

### Инициализация нового репозитория

```bash

mkdir my-project
```

## Файл .gitignore

Файл `.gitignore` указывает Git, какие файлы и папки игнорировать:

```gitignore
# Зависимости Node.js
node_modules/

# Логи
*.log
logs/

# Переменные окружения
.env
.env.local

# Папки сборки
dist/
build/

# Кэши
.cache/
.parcel-cache/

# IDE файлы

.idea/

*.swo

# Операционная система
.DS_Store
Thumbs.db

# Временные файлы
*.tmp
*.temp
```

## Основные команды Git

### `git status`

Показывает статус рабочей директории:

```bash
git status

# Краткий формат
git status -s
git status --short
```

Возможные состояния файлов:

- **Untracked** — новые файлы, не отслеживаемые Git
- **Modified** — измененные файлы
- **Staged** — файлы, добавленные в индекс (готовые к коммиту)

### `git add`

Добавление файлов в индекс (staging area):

```bash
# Добавить конкретный файл
git add index.html

# Добавить все файлы в текущей папке
git add .

# Добавить все файлы в проекте
git add -A
git add --all

# Добавить файлы по шаблону
git add *.js
git add src/

# Интерактивное добавление
git add -i
git add --interactive
```

### `git commit`

Фиксация изменений:

```bash
# Коммит с сообщением
git commit -m "Добавлен файл index.html"

# Коммит с подробным описанием
git commit -m "Добавлена главная страница" -m "Создан базовый HTML шаблон с навигацией"

# Коммит всех измененных файлов (пропуск git add)
git commit -am "Обновлены стили"

# Открытие редактора для написания сообщения
git commit
```

### Правила хороших сообщений коммитов

```bash
# ✅ Хорошо
git commit -m "Добавлена форма регистрации пользователей"
git commit -m "Исправлена ошибка валидации email"
git commit -m "Обновлены зависимости до последних версий"

# ❌ Плохо
git commit -m "фикс"
git commit -m "работает"
git commit -m "коммит"
```

## Просмотр истории

### `git log`

Просмотр истории коммитов:

```bash
# Полная история
git log

# Сокращенный формат
git log --oneline

# С графиком веток
git log --graph

# История конкретного файла
git log index.html

# Последние N коммитов
git log -5

# С показом изменений
git log -p

# Статистика изменений
git log --stat
```

### Красивое форматирование истории

```bash
# Кастомный формат
git log --pretty=format:"%h - %an, %ar : %s"

# С графиком и цветами
git log --graph --pretty=format:"%Cred%h%Creset -%C(yellow)%d%Creset %s %Cgreen(%cr) %C(bold blue)<%an>%Creset" --abbrev-commit

# Создание alias для красивого лога
git config --global alias.lg "log --graph --pretty=format:'%Cred%h%Creset -%C(yellow)%d%Creset %s %Cgreen(%cr) %C(bold blue)<%an>%Creset' --abbrev-commit"

# Теперь можно использовать
git lg
```

## Отмена изменений

### Отмена изменений в рабочей директории

```bash
# Отменить изменения в конкретном файле
git checkout -- index.html

# Отменить все изменения
git checkout -- .

# Современный способ (Git 2.23+)
git restore index.html
git restore .
```

### Отмена добавления в индекс

```bash
# Убрать файл из индекса
git reset HEAD index.html

# Убрать все файлы из индекса
git reset HEAD

# Современный способ
git restore --staged index.html
git restore --staged .
```

### Изменение последнего коммита

```bash
# Изменить сообщение последнего коммита
git commit --amend -m "Новое сообщение"

# Добавить файлы к последнему коммиту
git add забытый_файл.js
git commit --amend --no-edit
```

## Работа с удаленными репозиториями

### Добавление удаленного репозитория

```bash
# Добавить удаленный репозиторий
git remote add origin https://github.com/username/repository.git

# Просмотр удаленных репозиториев
git remote -v

# Переименование удаленного репозитория
git remote rename origin upstream

# Удаление удаленного репозитория
git remote remove origin
```

### `git push`

Отправка изменений в удаленный репозиторий:

```bash
# Первая отправка (установка upstream)
git push -u origin main

# Обычная отправка
git push

# Отправка в конкретную ветку
git push origin feature-branch

# Принудительная отправка (осторожно!)
git push --force
```

### `git pull`

Получение изменений из удаленного репозитория:

```bash
# Получение и слияние изменений
git pull

# Получение изменений из конкретной ветки
git pull origin main

# Получение с rebase
git pull --rebase
```

## Практический пример

### Создание первого проекта

```bash
# 1. Создание проекта
mkdir my-website
cd my-website

# 2. Инициализация Git
git init

# 3. Создание файлов
echo "# Мой сайт" > README.md
echo "<!DOCTYPE html><html><head><title>Мой сайт</title></head><body><h1>Добро пожаловать!</h1></body></html>" > index.html

# 4. Создание .gitignore
cat > .gitignore << EOF
node_modules/
.env
*.log
EOF

# 5. Добавление файлов
git add .

# 6. Первый коммит
git commit -m "Начальный коммит: добавлены основные файлы"

# 7. Просмотр статуса
git status

# 8. Просмотр истории
git log --oneline
```

### Внесение изменений

```bash
# 1. Изменение файла
echo "<p>Новый параграф</p>" >> index.html

# 2. Просмотр изменений
git status
git diff

# 3. Добавление и коммит
git add index.html
git commit -m "Добавлен новый параграф на главную страницу"

# 4. Создание CSS файла
cat > style.css << EOF
body {
    font-family: Arial, sans-serif;
    margin: 0;
    padding: 20px;
}

h1 {
    color: #333;
}
EOF

# 5. Добавление CSS
git add style.css
git commit -m "Добавлены базовые стили"
```

## Алиасы для удобства

```bash
# Создание полезных алиасов
git config --global alias.co checkout
git config --global alias.br branch
git config --global alias.ci commit
git config --global alias.st status
git config --global alias.unstage 'reset HEAD --'
git config --global alias.last 'log -1 HEAD'
git config --global alias.visual '!gitk'

# Теперь можно использовать сокращения
git st        # вместо git status
git co main   # вместо git checkout main
git ci -m "..." # вместо git commit -m "..."
```

## Полезные команды для анализа

```bash
# Показать изменения между коммитами
git diff HEAD~1 HEAD

# Показать файлы в коммите
git show --name-only abc123

# Поиск в истории коммитов
git log --grep="исправление"

# Поиск изменений в коде
git log -S "функция_поиска"

# Показать, кто изменял строки в файле
git blame index.html

# Статистика контрибьюторов
git shortlog -sn
```
