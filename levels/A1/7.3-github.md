# **7.3. Создание и управление удаленными репозиториями**

## Введение в GitHub

**GitHub** — это веб-платформа для хостинга Git-репозиториев, которая предоставляет дополнительные инструменты для совместной разработки, управления проектами и автоматизации.

### Регистрация на GitHub

1. Перейдите на [github.com](https://github.com)
2. Нажмите "Sign up"
3. Заполните форму регистрации
4. Подтвердите email адрес

### Настройка профиля

```bash
# Убедитесь, что ваши настройки Git совпадают с GitHub
git config --global user.name "Ваше имя на GitHub"
git config --global user.email "ваш_email@на_github.com"
```

## Создание удаленного репозитория

### Через веб-интерфейс GitHub

1. Войдите в GitHub
2. Нажмите кнопку "+" в правом верхнем углу
3. Выберите "New repository"
4. Заполните информацию:
   - **Repository name** — имя репозитория
   - **Description** — описание проекта
   - **Public/Private** — видимость репозитория
   - **Initialize with README** — создать README файл
   - **Add .gitignore** — выбрать шаблон .gitignore
   - **Choose a license** — выбрать лицензию

### Связывание локального репозитория с GitHub

#### Вариант 1: У вас уже есть локальный репозиторий

```bash
# Добавить удаленный репозиторий
git remote add origin https://github.com/username/repository-name.git

# Проверить удаленные репозитории
git remote -v

# Отправить код на GitHub
git push -u origin main
```

#### Вариант 2: Клонирование с GitHub

```bash
# Клонировать репозиторий
git clone https://github.com/username/repository-name.git

# Перейти в папку проекта
cd repository-name

# Проверить удаленные репозитории
git remote -v
```

## Аутентификация на GitHub

### Personal Access Token (рекомендуется)

1. Перейдите в Settings → Developer settings → Personal access tokens
2. Нажмите "Generate new token"
3. Выберите необходимые права доступа
4. Скопируйте токен (он больше не будет показан!)

```bash
# Использование токена вместо пароля
git clone https://username:token@github.com/username/repository.git

# Или настроить Git для хранения токена
git config --global credential.helper store
# При первом push введите username и токен вместо пароля
```

### SSH ключи (альтернативный способ)

```bash
# Генерация SSH ключа
ssh-keygen -t ed25519 -C "ваш_email@github.com"

# Добавление ключа в ssh-agent
eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ed25519

# Скопировать публичный ключ
cat ~/.ssh/id_ed25519.pub
```

Добавьте скопированный ключ в GitHub: Settings → SSH and GPG keys → New SSH key

```bash
# Проверка SSH соединения
ssh -T git@github.com

# Клонирование через SSH
git clone git@github.com:username/repository.git
```

## Основные операции с удаленным репозиторием

### `git push`

Отправка изменений на GitHub:

```bash
# Первая отправка (установка upstream)
git push -u origin main

# Обычная отправка
git push

# Отправка конкретной ветки
git push origin feature-branch

# Отправка всех веток
git push --all

# Отправка тегов
git push --tags

# Принудительная отправка (осторожно!)
git push --force
git push --force-with-lease  # более безопасная версия
```

### `git pull`

Получение изменений с GitHub:

```bash
# Получение и слияние изменений
git pull

# Получение из конкретной ветки
git pull origin main

# Получение с rebase вместо merge
git pull --rebase

# Получение без слияния
git fetch
```

### `git fetch`

Получение информации без слияния:

```bash
# Получить все изменения
git fetch

# Получить из конкретного remote
git fetch origin

# Получить конкретную ветку
git fetch origin main

# Получить и удалить несуществующие ветки
git fetch --prune
```

## Работа с ветками на GitHub

### Создание и отправка ветки

```bash
# Создать новую ветку
git checkout -b feature/new-functionality

# Внести изменения и зафиксировать
git add .
git commit -m "Добавлена новая функциональность"

# Отправить ветку на GitHub
git push -u origin feature/new-functionality
```

### Отслеживание удаленных веток

```bash
# Посмотреть все ветки (включая удаленные)
git branch -a

# Посмотреть только удаленные ветки
git branch -r

# Создать локальную ветку, отслеживающую удаленную
git checkout -b local-branch origin/remote-branch

# Установить отслеживание для существующей ветки
git branch --set-upstream-to=origin/main main
```

### Удаление веток

```bash
# Удалить локальную ветку
git branch -d feature-branch

# Принудительное удаление локальной ветки
git branch -D feature-branch

# Удалить удаленную ветку
git push origin --delete feature-branch
```

## Pull Requests (PR)

Pull Request — это запрос на включение ваших изменений в основную ветку проекта.

### Создание Pull Request

1. Отправьте вашу ветку на GitHub
2. Перейдите в репозиторий на GitHub
3. Нажмите "Compare & pull request"
4. Заполните информацию:
   - **Title** — краткое описание изменений
   - **Description** — подробное описание
   - **Reviewers** — кто должен просмотреть код
   - **Assignees** — ответственные за PR
   - **Labels** — метки для категоризации

### Лучшие практики для PR

```markdown
# Пример хорошего описания PR

## Что изменено

- Добавлена форма регистрации пользователей
- Реализована валидация email
- Добавлены тесты для новой функциональности

## Зачем

Реализация пользовательской регистрации согласно требованиям из issue #123

## Как тестировать

1. Перейти на страницу `/register`
2. Попробовать зарегистрироваться с некорректным email
3. Убедиться, что показывается ошибка валидации
4. Зарегистрироваться с корректными данными

## Скриншоты

[Добавить скриншоты интерфейса]

## Связанные issue

Closes #123
```

### Работа с PR через командную строку

```bash
# После создания PR и получения замечаний
git checkout feature-branch

# Внести исправления
git add .
git commit -m "Исправлены замечания по code review"

# Отправить обновления (PR автоматически обновится)
git push
```

## Issues (Задачи)

Issues используются для отслеживания задач, багов и предложений.

### Создание Issue

1. Перейдите на вкладку "Issues"
2. Нажмите "New issue"
3. Заполните:
   - **Title** — краткое описание проблемы
   - **Description** — подробное описание
   - **Labels** — метки (bug, enhancement, question и т.д.)
   - **Assignees** — ответственные
   - **Milestone** — этап проекта

### Связывание Issue с коммитами

```bash
# Коммиты могут автоматически закрывать issues
git commit -m "Исправлен баг с валидацией формы

Fixes #42"

# Другие ключевые слова:
# Closes #42, Resolves #42, Fixes #42
```

## Форки и контрибуция в открытые проекты

### Создание форка

1. Перейдите в репозиторий, в который хотите внести вклад
2. Нажмите кнопку "Fork" в правом верхнем углу
3. Выберите аккаунт, куда создать форк

### Работа с форком

```bash
# Клонировать ваш форк
git clone https://github.com/ваш-username/original-repo.git
cd original-repo

# Добавить оригинальный репозиторий как upstream
git remote add upstream https://github.com/original-owner/original-repo.git

# Проверить remote репозитории
git remote -v
# origin    https://github.com/ваш-username/original-repo.git (fetch)
# origin    https://github.com/ваш-username/original-repo.git (push)
# upstream  https://github.com/original-owner/original-repo.git (fetch)
# upstream  https://github.com/original-owner/original-repo.git (push)
```

### Синхронизация с upstream

```bash
# Получить изменения из оригинального репозитория
git fetch upstream

# Переключиться на main ветку
git checkout main

# Объединить изменения
git merge upstream/main

# Отправить обновления в ваш форк
git push origin main
```

### Создание PR из форка

1. Создайте ветку для новой функциональности
2. Внесите изменения и отправьте в ваш форк
3. Перейдите в оригинальный репозиторий
4. GitHub предложит создать PR из вашей ветки

## GitHub Pages

Бесплатный хостинг статических сайтов от GitHub.

### Настройка GitHub Pages

1. Перейдите в Settings репозитория
2. Найдите раздел "Pages"
3. Выберите источник:
   - **Deploy from a branch** — из ветки (обычно main)
   - **GitHub Actions** — через автоматизацию

### Публикация сайта

```bash
# Структура для GitHub Pages
index.html          # Главная страница
css/
  style.css
js/
  script.js
images/
  logo.png

# Коммит и push
git add .
git commit -m "Добавлен сайт для GitHub Pages"
git push origin main
```

Сайт будет доступен по адресу: `https://username.github.io/repository-name`

## Автоматизация с GitHub Actions

Базовый пример автоматического деплоя:

```yaml
# .github/workflows/deploy.yml
name: Deploy to GitHub Pages

on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v2

      - name: Setup Node.js
        uses: actions/setup-node@v2
        with:
          node-version: "16"

      - name: Install dependencies
        run: npm install

      - name: Build
        run: npm run build

      - name: Deploy
        uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./dist
```

## Управление релизами

### Создание релиза

```bash
# Создать тег
git tag -a v1.0.0 -m "Первый релиз"

# Отправить тег на GitHub
git push origin v1.0.0

git push --tags
```

1. Перейдите на вкладку "Releases"
2. Нажмите "Create a new release"
3. Выберите тег или создайте новый
4. Заполните описание релиза
5. Приложите файлы (если необходимо)
