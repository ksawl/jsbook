# **7.2. Журнал коммитов и откат изменений**

## Углубленная работа с `git log`

### Основные форматы вывода

```bash
# Полная информация
git log

# Сокращенный формат (одна строка на коммит)
git log --oneline

# С показом изменений
git log -p

# Статистика изменений
git log --stat

# График веток
git log --graph

# Комбинирование опций
git log --oneline --graph --decorate
```

### Фильтрация истории

#### По дате

```bash
# Коммиты за последний день
git log --since="1 day ago"

# Коммиты за последнюю неделю
git log --since="1 week ago"

# Коммиты между датами
git log --since="2023-01-01" --until="2023-12-31"

# Коммиты за конкретный день
git log --since="2023-09-18" --until="2023-09-19"
```

#### По автору

```bash
# Коммиты конкретного автора
git log --author="Иван Петров"

# Несколько авторов (регулярные выражения)
git log --author="Иван\|Мария"

# Исключение автора
git log --author="^(?!Бот).*"
```

#### По сообщению коммита

```bash
# Поиск по ключевому слову
git log --grep="исправление"

# Поиск нескольких слов (OR)
git log --grep="bug\|fix"

# Поиск с учетом регистра
git log --grep="Bug" -i
```

#### По изменениям в коде

```bash
# Поиск добавления/удаления строки кода
git log -S "функция_поиска"

# Поиск изменений в коде (регулярные выражения)
git log -G "функция.*поиск"

# Изменения в конкретном файле
git log -- index.html

# Изменения в папке
git log -- src/
```

### Кастомное форматирование

```bash
# Кастомный формат
git log --pretty=format:"%h - %an, %ar : %s"

# Полезные плейсхолдеры:
# %H - полный хэш коммита
# %h - сокращенный хэш коммита
# %T - хэш дерева
# %P - хэши родительских коммитов
# %an - имя автора
# %ae - email автора
# %ad - дата автора
# %ar - относительная дата автора
# %cn - имя коммиттера
# %ce - email коммиттера
# %cd - дата коммиттера
# %cr - относительная дата коммиттера
# %s - сообщение коммита
# %f - заголовок коммита (для имен файлов)

# Красивый формат с цветами
git log --pretty=format:"%C(red)%h%C(reset) - %C(yellow)%d%C(reset) %s %C(green)(%cr) %C(bold blue)<%an>%C(reset)"
```

### Ограничение количества результатов

```bash
# Последние 5 коммитов
git log -5

# Пропустить первые 10, показать следующие 5
git log --skip=10 -5

# Коммиты до определенного
git log abc123..

# Коммиты между двумя коммитами
git log abc123..def456
```

## Детальный анализ коммитов

### `git show`

Подробная информация о коммите:

```bash
# Показать последний коммит
git show

# Показать конкретный коммит
git show abc123

# Показать только файлы, которые изменились
git show --name-only abc123

# Показать статистику изменений
git show --stat abc123

# Показать изменения без diff
git show --pretty=fuller --no-patch abc123
```

### `git diff`

Сравнение изменений:

```bash
# Изменения в рабочей директории
git diff

# Изменения в индексе (staged)
git diff --staged
git diff --cached

# Между коммитами
git diff abc123 def456

# Между ветками
git diff main feature

# Изменения в конкретном файле
git diff abc123 -- index.html

# Статистика изменений
git diff --stat abc123 def456

# Только имена файлов
git diff --name-only abc123 def456
```

### `git blame`

Аннотация строк файла:

```bash
# Показать автора каждой строки
git blame index.html

# С номерами строк
git blame -n index.html

# Определенный диапазон строк
git blame -L 10,20 index.html

# Игнорировать пробелы
git blame -w index.html

# Показать email авторов
git blame -e index.html
```

## Способы отката изменений

### 1. Отмена изменений в рабочей директории

#### Современные команды (Git 2.23+)

```bash
# Восстановить файл из последнего коммита
git restore index.html

# Восстановить все файлы
git restore .

# Восстановить файл из конкретного коммита
git restore --source=abc123 index.html

# Восстановить файлы из индекса
git restore --staged index.html
```

#### Классические команды

```bash
# Отменить изменения в файле
git checkout -- index.html

# Отменить все изменения
git checkout -- .

# Восстановить файл из конкретного коммита
git checkout abc123 -- index.html
```

### 2. Отмена изменений в индексе

```bash
# Убрать файл из индекса (современный способ)
git restore --staged index.html

# Убрать все файлы из индекса
git restore --staged .

# Классический способ
git reset HEAD index.html
git reset HEAD
```

### 3. Откат коммитов

#### `git reset`

Перемещение указателя HEAD:

```bash
# Мягкий сброс (сохранить изменения в рабочей директории)
git reset --soft HEAD~1

# Смешанный сброс (по умолчанию, сохранить в рабочей директории)
git reset HEAD~1
git reset --mixed HEAD~1

# Жесткий сброс (удалить все изменения)
git reset --hard HEAD~1

# Сброс к конкретному коммиту
git reset --hard abc123
```

**Различия между типами reset:**

- `--soft`: только перемещает HEAD, изменения остаются в индексе
- `--mixed` (по умолчанию): перемещает HEAD и сбрасывает индекс
- `--hard`: перемещает HEAD, сбрасывает индекс и рабочую директорию

#### `git revert`

Создание нового коммита, отменяющего изменения:

```bash
# Отменить последний коммит
git revert HEAD

# Отменить конкретный коммит
git revert abc123

# Отменить несколько коммитов
git revert abc123..def456

# Отменить без создания коммита (для дальнейшего редактирования)
git revert --no-commit abc123
```

**Разница между reset и revert:**

- `reset` изменяет историю (опасно для опубликованных коммитов)
- `revert` создает новый коммит отмены (безопасно для опубликованных коммитов)

### 4. Переключение на предыдущие коммиты

```bash
# Временно переключиться на старый коммит
git checkout abc123

# Вернуться к текущей ветке
git checkout main

# Создать новую ветку от старого коммита
git checkout -b fix-branch abc123
```

## Продвинутые техники

### Интерактивный rebase

```bash
# Изменить последние 3 коммита
git rebase -i HEAD~3

# В открывшемся редакторе можно:
# pick - оставить коммит как есть
# reword - изменить сообщение коммита
# edit - остановиться для редактирования
# squash - объединить с предыдущим коммитом
# drop - удалить коммит
```

### Cherry-pick

Применение конкретного коммита:

```bash
# Применить коммит из другой ветки
git cherry-pick abc123

# Применить несколько коммитов
git cherry-pick abc123 def456

# Применить без коммита
git cherry-pick --no-commit abc123
```

### Stash (временное сохранение)

```bash
# Сохранить текущие изменения
git stash

# Сохранить с сообщением
git stash save "работа над функцией X"

# Посмотреть список stash
git stash list

# Применить последний stash
git stash pop

# Применить конкретный stash
git stash apply stash@{1}

# Показать изменения в stash
git stash show -p stash@{0}

# Удалить stash
git stash drop stash@{1}

# Очистить все stash
git stash clear
```

## Восстановление потерянных коммитов

### `git reflog`

Журнал всех операций с HEAD:

```bash
# Показать все операции
git reflog

# Последние 10 операций
git reflog -10

# Восстановить коммит
git reset --hard HEAD@{2}

# Создать ветку от потерянного коммита
git branch lost-branch HEAD@{3}
```

### `git fsck`

Проверка целостности и поиск потерянных объектов:

```bash
# Найти висячие коммиты
git fsck --lost-found

# Подробная информация
git fsck --full
```

## Практические сценарии

### Сценарий 1: Исправление сообщения последнего коммита

```bash
# Неправильное сообщение
git commit -m "добавил функци"

# Исправление
git commit --amend -m "добавил функцию поиска"
```

### Сценарий 2: Добавление файла к последнему коммиту

```bash
# Забыли добавить файл
git add забытый_файл.js
git commit --amend --no-edit
```

### Сценарий 3: Откат опубликованного коммита

```bash
# НЕ используйте reset для опубликованных коммитов
# Используйте revert
git revert abc123
git push origin main
```

### Сценарий 4: Временная отмена изменений

```bash
# Нужно быстро переключиться на другую задачу
git stash

# Работа над другой задачей
git checkout other-branch
# ... работа ...
git checkout main

# Возврат к прежней работе
git stash pop
```

### Сценарий 5: Объединение нескольких коммитов

```bash
# Есть 3 мелких коммита, которые нужно объединить
git rebase -i HEAD~3

# В редакторе:
# pick abc123 первый коммит
# squash def456 второй коммит
# squash ghi789 третий коммит
```

### Сценарий 6: Исправление коммита в середине истории

```bash
# Найти проблемный коммит
git log --oneline

# Интерактивный rebase
git rebase -i abc123^

# Отметить коммит как edit
# После остановки внести изменения
git add .
git commit --amend
git rebase --continue
```

## Создание полезных алиасов

```bash
# Красивый лог
git config --global alias.lg "log --color --graph --pretty=format:'%Cred%h%Creset -%C(yellow)%d%Creset %s %Cgreen(%cr) %C(bold blue)<%an>%Creset' --abbrev-commit"

# Последний коммит
git config --global alias.last "log -1 HEAD"

# Отмена последнего коммита
git config --global alias.undo "reset HEAD~1 --mixed"

# Статус в кратком виде
git config --global alias.s "status -s"

# Показать изменения
git config --global alias.d "diff"

# Показать staged изменения
git config --global alias.ds "diff --staged"
```
