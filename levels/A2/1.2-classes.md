# **Классы**

Классы в JavaScript — это синтаксический сахар над прототипным наследованием. Они предоставляют более удобный и понятный способ создания объектов с общими свойствами и методами. Классы были введены в стандарте ECMAScript 2015 (ES6) и стали основным инструментом для организации кода в стиле объектно-ориентированного программирования (ООП).

## |1| **Синтаксис классов**

Класс объявляется с помощью ключевого слова `class`. Основными элементами класса являются **конструктор** и **методы**.

### 1. **Объявление класса**

```javascript
class Person {
  // Конструктор - вызывается при создании нового объекта
  constructor(name, age) {
    this.name = name;
    this.age = age;
  }

  // Метод класса - автоматически попадает в прототип
  greet() {
    console.log(`Привет, меня зовут ${this.name}, мне ${this.age} лет.`);
  }
}

// Создание экземпляра (инстанса) класса
const alice = new Person('Alice', 25);
alice.greet(); // Привет, меня зовут Alice, мне 25 лет.
```

### 2. **Почему это "синтаксический сахар"?**

Под капотом классы по-прежнему используют функции и прототипы. Создание класса выше почти эквивалентно следующему коду:

```javascript
function Person(name, age) {
  this.name = name;
  this.age = age;
}

Person.prototype.greet = function() {
  console.log(`Привет, меня зовут ${this.name}`);
};
```

**Особенности классов:**
- **Strict Mode**: Весь код внутри классов работает в строгом режиме автоматически.
- **Hoisting**: Классы не "всплывают". Вы не можете использовать класс до того, как объявили его.
- **Вызов только с `new`**: Класс нельзя вызвать как обычную функцию.

## |2| **Наследование**

Наследование позволяет создавать новые классы на основе существующих, расширяя или переопределяя их функциональность. Используется ключевое слово `extends`.

```javascript
class Animal {
  constructor(name) {
    this.name = name;
  }

  speak() {
    console.log(`${this.name} издает звук.`);
  }
}

class Dog extends Animal {
  constructor(name, breed) {
    super(name); // Вызов конструктора родителя (Animal)
    this.breed = breed;
  }

  // Переопределение метода
  speak() {
    console.log(`${this.name} гавкает (порода: ${this.breed}).`);
  }
}

const rex = new Dog('Рекс', 'Овчарка');
rex.speak(); // Рекс гавкает (порода: Овчарка).
```

- **`super()`**: Обязательно должен быть вызван в конструкторе потомка перед использованием `this`.
- **Наследование методов**: Потомок автоматически получает доступ ко всем методам родителя.

## |3| **Геттеры и сеттеры**

Геттеры и сеттеры позволяют контролировать доступ к свойствам объекта, выполняя дополнительную логику при их чтении или записи.

```javascript
class User {
  constructor(firstName, lastName) {
    this.firstName = firstName;
    this.lastName = lastName;
  }

  // Геттер - вычисляемое свойство
  get fullName() {
    return `${this.firstName} ${this.lastName}`;
  }

  // Сеттер - валидация и обработка данных
  set fullName(value) {
    const parts = value.split(' ');
    if (parts.length < 2) {
      console.error('Ошибка: неверный формат имени');
      return;
    }
    this.firstName = parts[0];
    this.lastName = parts[1];
  }
}

const user = new User('Иван', 'Иванов');
console.log(user.fullName); // Иван Иванов
user.fullName = 'Пётр Петров';
console.log(user.firstName); // Пётр
```

## |4| **Приватные поля и методы**

Начиная с современных стандартов (ES2022+), в JavaScript появились **настоящие** приватные свойства, которые начинаются с символа `#`. Они недоступны за пределами класса.

```javascript
class BankAccount {
  #balance = 0; // Приватное поле

  constructor(owner) {
    this.owner = owner;
  }

  deposit(amount) {
    if (amount > 0) {
      this.#balance += amount;
      this.#log(`Пополнение: ${amount}`);
    }
  }

  // Приватный метод
  #log(msg) {
    console.log(`[LOG] ${this.owner}: ${msg}`);
  }

  get balance() {
    return this.#balance;
  }
}

const account = new BankAccount('Николай');
account.deposit(500);
console.log(account.balance); // 500
// console.log(account.#balance); // Ошибка: приватное поле
```

## |5| **Статические методы и свойства**

Статические элементы принадлежат самому классу, а не его экземплярам. Они полезны для реализации общих функций (утилит).

```javascript
class MathHelper {
  static PI = 3.14159;

  static calculateCircleArea(radius) {
    return this.PI * (radius ** 2);
  }
}

console.log(MathHelper.PI); // 3.14159
console.log(MathHelper.calculateCircleArea(10)); // 314.159

const helper = new MathHelper();
// helper.calculateCircleArea(10); // Ошибка: метод не в экземпляре
```

## **Итог**

- **Классы** — это удобный синтаксис для работы с прототипами.
- **`constructor`** используется для инициализации свойств.
- **`extends`** и **`super`** обеспечивают наследование.
- **Геттеры и сеттеры** (`get`/`set`) создают "умные" свойства.
- **Приватные поля** (`#`) защищают внутренние данные от внешнего вмешательства.
- **Статические элементы** (`static`) хранятся в самом классе.

## **Практика**

### 1. **Класс "Умный Светильник"**
Создайте класс `Lamp`, который имеет:
- Приватное поле `#isOn` (по умолчанию `false`).
- Приватное поле `#brightness` (от 0 до 100).
- Геттер `status`, возвращающий строку "Включен (яркость: X%)" или "Выключен".
- Метод `toggle()`, переключающий состояние.
- Сеттер `brightness`, который не позволяет установить значение меньше 0 или больше 100.

### 2. **Наследование "Транспорт"**
1. Создайте базовый класс `Vehicle` с конструктором (марка, год) и методом `getInfo()`.
2. Создайте класс `Car`, который наследует от `Vehicle` и добавляет свойство `doors`.
3. Переопределите `getInfo()`, чтобы он включал количество дверей.

### 3. **Статический Калькулятор**
Создайте класс `Geometry`, у которого будут статические методы для вычисления площади квадрата и прямоугольника.
