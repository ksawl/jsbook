# **Метапрограммирование: Символы, Proxy и Reflect**

**Метапрограммирование** — это способ написания программ, которые могут анализировать или изменять другие программы (или даже самих себя). В JavaScript основными инструментами для этого являются **Символы**, объект **Proxy** и объект **Reflect**.

## |1| **Продвинутые возможности Символов**

Мы уже знакомы с **Symbol** как способом создания уникальных ключей. На уровне C1 мы углубимся в глобальный реестр и "общеизвестные" символы.

### **Глобальный реестр: `Symbol.for`**

Иногда нужно, чтобы символы с одинаковым именем были одинаковыми в разных частях приложения (например, в разных модулях или даже фреймах).

- `Symbol.for(key)` — ищет существующий символ с этим ключом и возвращает его. Если нет — создаёт.
- `Symbol.keyFor(sym)` — возвращает имя глобального символа.

```javascript
const s1 = Symbol.for("app.id");
const s2 = Symbol.for("app.id");

console.log(s1 === s2); // true
console.log(Symbol.keyFor(s1)); // "app.id"
```

### **Хорошо известные символы (Well-known Symbols)**

Это системные символы, через которые можно менять внутреннее поведение объектов:
- `Symbol.toPrimitive`: Управляет преобразованием объекта в строку/число.
- `Symbol.toStringTag`: Меняет строку, возвращаемую `Object.prototype.toString`.
- `Symbol.hasInstance`: Настраивает поведение оператора `instanceof`.

## |2| **Объект Proxy: Мастер перехвата**

**Proxy** — это "обёртка" вокруг объекта, которая позволяет перехватывать и изменять фундаментальные операции (чтение свойства, запись, удаление и т.д.).

```javascript
const user = { name: "Alice" };

const proxy = new Proxy(user, {
    get(target, prop, receiver) {
        console.log(`Чтение свойства: ${prop}`);
        return prop in target ? target[prop] : "Свойство отсутствует";
    },
    set(target, prop, value) {
        if (prop === "age" && value < 0) throw new Error("Возраст не может быть отрицательным");
        target[prop] = value;
        return true; // Обязательно вернуть true при успехе
    }
});

console.log(proxy.name); // Чтение свойства: name -> "Alice"
console.log(proxy.age);  // "Свойство отсутствует"
```

### **Популярные ловушки (Traps):**
- `has(target, prop)` — для оператора `in`.
- `deleteProperty(target, prop)` — для оператора `delete`.
- `ownKeys(target)` — для `Object.keys()` и `for...in`.

### **Продвинутые паттерны Proxy**

На уровне C1 Proxy используется не просто для логирования, а для изменения архитектурного поведения объектов.

#### **1. Реактивность (Vue-style)**
Этот паттерн позволяет автоматически реагировать на любые изменения данных.

```javascript
function reactive(target, callback) {
    return new Proxy(target, {
        set(target, prop, value) {
            const oldValue = target[prop];
            target[prop] = value;
            callback(prop, oldValue, value);
            return true;
        }
    });
}

const state = reactive({ count: 0 }, (prop, oldV, newV) => {
    console.log(`Изменено свойство ${prop}: ${oldV} -> ${newV}`);
});

state.count++; // Изменено свойство count: 0 -> 1
```

#### **2. Скрытые свойства (Private Access Control)**
Proxy может полностью скрыть свойства (например, начинающиеся с `_`), сделав их невидимыми для `in`, `Object.keys()` и даже прямого доступа.

```javascript
const withHiddenProps = (target, prefix = '_') => {
    return new Proxy(target, {
        has: (obj, prop) => prop in obj && !prop.startsWith(prefix),
        ownKeys: (obj) => Reflect.ownKeys(obj).filter(key => typeof key !== 'string' || !key.startsWith(prefix)),
        get: (obj, prop) => prop.startsWith?.(prefix) ? undefined : obj[prop]
    });
};

const user = withHiddenProps({ name: "Alice", _id: "secret-123" });
console.log("_id" in user); // false
console.log(Object.keys(user)); // ["name"]
console.log(user._id); // undefined
```

#### **3. Умная индексация (IndexedArray)**
Мы можем создать массив, который ведет себя как обычный массив, но умеет мгновенно находить элементы по `id` (O(1)).

```javascript
const IndexedArray = new Proxy(Array, {
    construct(target, [args]) {
        const index = {};
        args.forEach(item => index[item.id] = item);

        return new Proxy(new target(...args), {
            get(arr, prop) {
                if (prop === 'findById') return id => index[id];
                if (prop === 'push') return item => {
                    index[item.id] = item;
                    return arr.push(item);
                };
                return arr[prop];
            }
        });
    }
});

const users = new IndexedArray([{ id: 10, name: "Ivan" }]);
console.log(users.findById(10)); // { id: 10, name: "Ivan" }
```

## |3| **Ловушки для функций и классов**

Proxy может оборачивать не только объекты, но и функции (включая конструкторы).

### **Ловушка `apply`**
Перехватывает вызов функции.

```javascript
function sum(a, b) { return a + b; }

const logSum = new Proxy(sum, {
    apply(target, thisArg, args) {
        console.log(`Вызов с аргументами: ${args}`);
        return target.apply(thisArg, args) * 10; // Можем изменить результат
    }
});

console.log(logSum(2, 3)); // Вызов с аргументами: 2,3 -> 50
```

### **Ловушка `construct`**
Перехватывает создание экземпляра через `new`.

```javascript
class User {}

const UserProxy = new Proxy(User, {
    construct(target, args) {
        console.log("Создание нового пользователя...");
        return new target(...args);
    }
});

const user = new UserProxy(); // Создание нового пользователя...
```

## |4| **Объект Reflect: Зеркало операций**

**Reflect** — это встроенный объект, методы которого зеркально повторяют ловушки Proxy. 

**Зачем он нужен?**
1. **Упрощение проксирования**: Вместо `target[prop] = value` внутри ловушки лучше использовать `Reflect.set(...)`. Это возвращает `true/false` вместо того, чтобы выбрасывать исключение, и корректно работает с прототипами.
2. **Функциональный стиль**: Операции вроде `delete obj.prop` превращаются в `Reflect.deleteProperty(obj, 'prop')`.

```javascript
const proxy = new Proxy(user, {
    get(target, prop, receiver) {
        // Правильный способ проброса вызова
        return Reflect.get(target, prop, receiver);
    }
});
```

## |5| **Динамическое создание классов**

Благодаря тому, что классы в JS — это функции, мы можем создавать их динамически.

### **Способ 1: `new Function`**
Позволяет собрать класс буквально из строки кода (используйте осторожно!).

```javascript
const createClass = (name) => {
    return new Function(`return class ${name} { constructor(v) { this.v = v; } }`)();
};

const MyClass = createClass("DynamicHero");
const hero = new MyClass(100);
console.log(hero.constructor.name); // "DynamicHero"
```

### **Способ 2: Класс-выражение**
```javascript
function makeClass(phrase) {
  return class {
    sayHi() { console.log(phrase); }
  };
}

const User = makeClass("Привет!");
new User().sayHi(); // "Привет!"
```

## **Итог**

- **Symbol.for** создает глобальные символы, доступные во всем приложении.
- **Proxy** позволяет создать "умную" обертку, которая может валидировать, логировать или даже запрещать операции над объектом.
- **Reflect** предоставляет стандартные методы для выполнения тех же операций, которые перехватывает Proxy.
- **Метапрограммирование** делает твой код гибким и мощным, позволяя создавать библиотеки (как Vue или MobX), которые "магически" реагируют на изменения данных.

## **Практика**

### 1. **Proxy-валидатор "Только строки"**
Создайте Proxy для объекта, который разрешает записывать в свойства только строковые значения. Если передано число или другой тип — выбрасывайте ошибку.

### 2. **Спрятанные свойства (Private-like)**
Реализуйте функцию `withHiddenProps(obj, prefix)`, которая возвращает Proxy. Этот Proxy должен скрывать все свойства, начинающиеся с `prefix` (например, `_`), от оператора `in` и метода `Object.keys()`.

### 3. **Безопасное чтение (Safe Get)**
Создайте Proxy, который при обращении к несуществующему свойству не возвращает `undefined`, а выдает ошибку: `Property [name] does not exist`.
