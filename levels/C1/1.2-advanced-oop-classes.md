## |1| **Инкапсуляция: Четыре способа защиты данных**

**Инкапсуляция** — это сокрытие внутренней реализации. В JavaScript она прошла долгую эволюцию.

### **1. Замыкания (Closures)**
Классический способ до появления классов. Данные полностью недоступны извне — это «жесткая» приватность.

```javascript
function createAccount(initialBalance) {
    let _balance = initialBalance; // Настоящая приватная переменная
    return {
        deposit: (amount) => { _balance += amount; },
        getBalance: () => _balance
    };
}
```

### **2. Приватные поля (ES2022)**
Современный стандарт с использованием символа `#`.

```javascript
class BankAccount {
    #balance = 0; // Приватное поле

    deposit(amount) {
        this.#balance += amount;
    }
}
```

### **3. WeakMap (Паттерн для библиотек)**
Позволяет хранить приватные данные «сбоку» от объекта, обеспечивая их автоматическое удаление сборщиком мусора.

```javascript
const privateData = new WeakMap();

class User {
    constructor(id) {
        privateData.set(this, { internalId: id });
    }
    getInternalId() {
        return privateData.get(this).internalId;
    }
}
```

### **4. Символы (Symbols)**
Создают «полуприватные» свойства. Они не видны в `Object.keys()`, но их можно достать через `getOwnPropertySymbols`.

## |2| **Аксессоры (get/set): Умные свойства**

Геттеры и сеттеры в C1 используются не просто для доступа, а для **валидации**, **логирования** и **ленивых вычислений**.

```javascript
class Temperature {
    constructor(celsius) {
        this._celsius = celsius;
    }

    get fahrenheit() {
        console.log("Вычисляем Фаренгейт...");
        return this._celsius * 1.8 + 32;
    }

    set fahrenheit(value) {
        if (value < -459.67) throw new Error("Ниже абсолютного нуля!");
        this._celsius = (value - 32) / 1.8;
    }
}
```

## |3| **Динамическое создание классов и фабрики**

В JavaScript классы — это «первоклассные граждане» (First-class citizens). Мы можем передавать их в функции и возвращать из них.

### **Класс-выражение как результат функции**
Это основа паттерна «Миксины» (Mixins).

```javascript
function makeClass(greeting) {
    return class {
        sayHi() { console.log(greeting); }
    };
}

const HelloClass = makeClass("Привет, Харизма!");
new HelloClass().sayHi();
```

### **Фабричные методы**
Вместо прямого `new` мы используем статические методы для более чистого API.

```javascript
class User {
    constructor(name, role) {
        this.name = name;
        this.role = role;
    }
    static createAdmin(name) { return new User(name, 'admin'); }
}
```

## |4| **Философия ООП: Наследование vs Композиция**

> [!IMPORTANT]
> На уровне C1 важно понимать: глубокое наследование («хрупкий базовый класс») — это антипаттерн. 

### **Наследование (Relationship: "is-a")**
Используйте только тогда, когда один класс *является* подвидом другого.
`Eagle extends Bird` — хорошо. `User extends Database` — плохо.

### **Композиция (Relationship: "has-a")**
Вместо того чтобы наследоваться от всего подряд, объект *содержит* в себе другие объекты.

```javascript
class Robot {
    constructor() {
        this.motor = new Motor();
        this.camera = new Camera();
    }
}
```

---

## **Итог**

- **Инкапсуляция** защищает целостность объекта. Используйте `#private` для простоты или `WeakMap` для гибкости.
- **Геттеры/Сеттеры** превращают свойства в «живую» логику.
- **Динамические классы** позволяют строить гибкие архитектуры (плагины, миксины).
- **Композиция** предпочтительнее наследования в сложных системах.

## **Практика**

### 1. **Валидатор через WeakMap**
Реализуйте класс `SecureData`, который хранит данные в `WeakMap`. При попытке записи через сеттер проверяйте тип данных (разрешены только объекты).

### 2. **Динамическая Фабрика**
Создайте функцию `createModel(className, schema)`, которая возвращает класс с заданным именем и геттерами/сеттерами для каждого поля, указанного в `schema`.

### 3. **Миксин логирования**
Напишите функцию `withLogging(BaseClass)`, которая возвращает новый класс, расширяющий `BaseClass` и логирующий вызовы всех его методов.
