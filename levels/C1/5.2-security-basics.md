## |1| **Хеширование паролей: bcrypt и argon2**

Никогда не храните пароли в открытом виде! Даже если база данных попадет в руки злоумышленников, они не должны увидеть реальные пароли.

### **Почему не md5 или sha256?**
Эти алгоритмы слишком быстрые. Современные видеокарты могут перебирать миллиарды таких хешей в секунду (brute-force).

### **bcrypt и argon2**
Это алгоритмы, специально созданные медленными. Они используют **Salt** (соль) — случайную строку, которая добавляется к паролю перед хешированием. Это защищает от использования "радужных таблиц" (баз готовых хешей).

- **bcrypt**: Стандарт де-факто для Node.js приложений.
- **argon2**: Победитель Password Hashing Competition, считается самым современным и стойким (но требует больше ресурсов).

```javascript
const bcrypt = require('bcrypt');
const saltRounds = 10;
const hash = await bcrypt.hash('my_password', saltRounds);
// Проверка пароля:
const match = await bcrypt.compare('input_password', hash);
```

## |2| **CORS: Cross-Origin Resource Sharing**

**CORS** — это механизм безопасности браузера, который ограничивает запросы с одного домена (origin) на другой.

### **Как это работает?**
Когда браузер делает запрос на другой домен, он отправляет заголовок `Origin`. Сервер должен ответить заголовком `Access-Control-Allow-Origin`, указав, разрешен ли этот домен.

### **Preflight запросы (OPTIONS)**
Для небезопасных методов (`PUT`, `DELETE`, `POST` с JSON) браузер сначала отправляет "пробный" запрос `OPTIONS`, чтобы узнать разрешения сервера.

> [!TIP]
> В Node.js (Express) лучше всего использовать мидлвар `cors` и настраивать белый список (whitelist) разрешенных доменов вместо использования `*`.

## |3| **HTTPS и шифрование данных**

**HTTPS** (HTTP Secure) — это расширение протокола HTTP, которое шифрует все данные с помощью **TLS** (Transport Layer Security).

### **Что защищает HTTPS?**
1. **Конфиденциальность**: Данные (пароли, куки) не могут быть перехвачены провайдером или хакером в публичном Wi-Fi.
2. **Целостность**: Данные не могут быть незаметно изменены в пути.
3. **Аутентификация**: Вы уверены, что общаетесь именно с нужным сервером, а не с его имитацией.

> [!IMPORTANT]
> На уровне C1 использование HTTPS обязательно не только для продакшена, но и для корректной работы многих браузерных API (Service Workers, Geolocation).

## **Итог**

- Используйте **bcrypt** или **argon2** с солью для хранения паролей.
- Настраивайте **CORS** явно, разрешая только нужные домены.
- Всегда используйте **HTTPS** для защиты трафика пользователей.
- Безопасность — это комплекс мер, а не одна "секретная кнопка".
