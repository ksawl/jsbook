# **Продвинутые регулярные выражения**

На уровне C1 мы выходим за рамки простого поиска слов и переходим к **метапрограммированию строк**. Продвинутые возможности RegExp позволяют не просто находить данные, но и анализировать контекст вокруг них, давать группам осмысленные имена и работать с текстом на уровне профессионального парсера.

## |1| **Именованные группы захвата**

Обычно группы захвата нумеруются: `\1`, `\2` и т.д. В сложном выражении легко запутаться в индексах. Именованные группы позволяют обращаться к частям совпадения по ключу.

**Синтаксис:** `(?<name>pattern)`

```javascript
const dateStr = "2023-10-25";
const regex = /(?<year>\d{4})-(?<month>\d{2})-(?<day>\d{2})/;

const result = dateStr.match(regex);
console.log(result.groups.year);  // "2023"
console.log(result.groups.month); // "10"
```

### **Использование в `replace`**
Именованные группы делают код замены гораздо более читаемым:

```javascript
const result = "2023-10-25".replace(regex, "$<day>.$<month>.$<year>");
console.log(result); // "25.10.2023"
```

## |2| **Проверки вперед и назад (Lookaround)**

Это "незахватывающие" проверки контекста. Они позволяют найти строку только в том случае, если перед ней или после нее **есть** или **нет** определенного текста.

### **1. Lookahead (Вперед)**
- `(?=pattern)` — Положительная: "есть после".
- `(?!pattern)` — Отрицательная: "нет после".

```javascript
// Найти число, только если за ним идет символ "$"
console.log("100$ и 500€".match(/\d+(?=\$)/g)); // ["100"]
```

### **2. Lookbehind (Назад)**
- `(?<=pattern)` — Положительная: "есть перед".
- `(?<!pattern)` — Отрицательная: "нет перед".

```javascript
// Найти число, только если перед ним НЕТ знака "-"
console.log("10 -5 100".match(/(?<!-)\b\d+/g)); // ["10", "100"]
```

## |3| **Продвинутые флаги: s, y, d**

### **Флаг `s` (dotAll)**
По умолчанию точка `.` не соответствует символу переноса строки. Флаг `s` заставляет её включать и `\n`.

```javascript
const str = "Привет\nМир";
console.log(/Привет.Мир/.test(str));  // false
console.log(/Привет.Мир/s.test(str)); // true
```

### **Флаг `y` (sticky)**
Поиск начинается строго с позиции `lastIndex` и не ищет дальше по строке. Удобно для создания лексических анализаторов.

### **Флаг `d` (indices)**
Добавляет в результат массив `indices`, который содержит точные координаты (начало и конец) каждой найденной группы.

## |4| **Сложные примеры парсинга**

Регулярные выражения на уровне C1 часто используются для разбора структурированных данных, где стандартные методы бессильны.

**Пример: Парсинг URL**
```javascript
const url = "https://example.com:8080/path?query=1";
const urlRegex = /^(?<proto>\w+):\/\/(?<host>[^:/]+)(?::(?<port>\d+))?(?<path>[^?#]*)?/;

const parts = url.match(urlRegex).groups;
console.log(parts); // { proto: "https", host: "example.com", port: "8080", path: "/path" }
```

## **Итог**

- **Именованные группы** избавляют от "магических чисел" индексов (\1, \2).
- **Lookaround** позволяет строить условия "если перед" или "если после", не включая эти части в результат.
- Флаги **s, y, d** дают тонкий контроль над поведением движка RegExp.
- Регулярные выражения — это микро-язык. Используйте комментарии (`//`) или разбивайте сложные выражения на части, если они становятся нечитаемыми.

## **Практика**

### 1. **Валидация пароля**
Напишите регулярное выражение с использованием **Lookahead**, которое проверяет пароль на соответствие условиям:
- Минимум 8 символов.
- Хотя бы одна заглавная буква.
- Хотя бы одна цифра.

### 2. **Парсинг цен**
Дана строка: `«Цена товара: 100 руб, скидка: 20$, доставка: 5 евро»`.
С помощью **Lookbehind** и **именованных групп** извлеките все суммы и их валюты в массив объектов `[{ amount: "100", currency: "руб" }, ...]`.

### 3. **Скрытые теги**
Используя флаг `s`, найдите содержимое между тегами `[code]...[/code]`, даже если внутри есть переносы строк.
