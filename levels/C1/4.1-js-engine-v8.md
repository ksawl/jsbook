## |1| **Архитектура V8: Ignition и TurboFan**

V8 не просто интерпретирует код строка за строкой. Он использует **JIT-компиляцию** (Just-In-Time).

### **Конвейер выполнения:**
1.  **Parser**: Превращает исходный код в **AST** (Abstract Syntax Tree).
2.  **Ignition**: Быстрый интерпретатор, который генерирует байт-код из AST. Код начинает выполняться мгновенно.
3.  **TurboFan**: Оптимизирующий компилятор. Он следит за часто выполняемым ("горячим") кодом и переводит его в высокоэффективный машинный код, учитывая типы данных, которые реально проходят через функции.

> [!TIP]
> Если типы данных в функции постоянно меняются (полиморфизм), TurboFan выполняет **деоптимизацию** и возвращает код обратно в Ignition. Стабильность типов — залог скорости.

## |2| **Скрытые классы (Hidden Classes)**

JavaScript — динамический язык, но на низком уровне процессору нужно знать точное смещение свойств в памяти. V8 решает это через **Hidden Classes** (в V8 они называются `Shapes`).

Когда вы создаете объект и добавляете свойства, V8 создает цепочку скрытых классов:
```javascript
function Point(x, y) {
  this.x = x; // Создается Класс_1 (свойство x)
  this.y = y; // Создается Класс_2 (свойства x и y)
}

const p1 = new Point(1, 2);
const p2 = new Point(3, 4); 
// p1 и p2 разделяют один и тот же скрытый класс. Это ОЧЕНЬ быстро.
```

> [!WARNING]
> Если вы меняете порядок инициализации свойств, V8 создаст разные скрытые классы, и оптимизация сломается. Всегда инициализируйте свойства в конструкторе в одном порядке.

## |3| **Встроенные кэши (Inline Caching)**

V8 запоминает, где именно в памяти находилось свойство при прошлом вызове функции. Если скрытый класс объекта совпадает с предыдущим, V8 пропускает поиск свойства и берет его по готовому адресу.

---

## |4| **Сборка мусора (Garbage Collection)**

V8 использует поколенческую стратегию сборки мусора (Generational GC). Память делится на две основные зоны:

### **1. Young Generation (New Space)**
Здесь рождаются все новые объекты. Она маленькая (1–8 МБ) и очищается очень часто методом **Scavenge**.
- Объекты, пережившие две очистки, "выживают" и переносятся в Old Space.

### **2. Old Generation (Old Space)**
Здесь живут долгоживущие объекты. Когда места становится мало, запускается **Major GC** (Mark-Sweep-Compact). Он останавливает выполнение кода (Stop-the-world), поэтому V8 старается делать это инкрементально.

### **WeakRef и FinalizationRegistry (ES2021)**
Позволяют держать "слабые" ссылки на объекты, не препятствуя их удалению сборщиком мусора.
```javascript
let cache = new WeakMap();
// Объект удалится из памяти, даже если он есть в WeakMap ключом.
```

## **Итог**

- **Пишите мономорфный код**: Передавайте в функции объекты одинаковой структуры.
- **Порядок важен**: Инициализируйте поля класса всегда в одном и том же порядке.
- **Избегайте `delete`**: Удаление свойств ломает скрытые классы объекта. Лучше присвоить `null`.
- **Понимайте GC**: Не создавайте лишних объектов в "горячих" циклах, чтобы не нагружать New Space сборщика.

## **Практика**

### 1. **Тест на скрытые классы**
Создайте два типа объектов: один через конструктор, другой через литерал с другим порядком свойств. Замерьте время доступа к свойствам в цикле на 10 млн итераций.

### 2. **Слабые ссылки**
Реализуйте простейшую систему кэширования изображений (представьте в виде строк имен файлов) с использованием `WeakMap`. Объясните, почему это лучше обычного `Map` для данной задачи.

### 3. **Анализ JIT**
Напишите функцию, которая сначала работает с числами, а потом (через несколько тысяч вызовов) начинает получать строки. Опишите, что произойдет в TurboFan в этот момент.
