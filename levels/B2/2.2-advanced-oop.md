## |1| **Инкапсуляция и уровни доступа**

Инкапсуляция позволяет защитить внутренние данные объекта от прямого изменения. В JavaScript есть три способа пометить свойство как "не для всех":

1.  **Приватные поля (`#balance`)**: Защищены на уровне движка. К ним нельзя обратиться снаружи — будет ошибка.
2.  **Защищенные поля (`_balance`)**: Соглашение между разработчиками. Технически доступны, но сигнализируют: "не трогай это напрямую".
3.  **Замыкания**: Использование области видимости функции для скрытия переменной (классический способ).

```javascript
class BankAccount {
    #balance = 0; // Настоящая приватность

    deposit(amount) {
        if (amount > 0) this.#balance += amount;
    }

    get balance() { return this.#balance; }
}
```

## |2| **Статические свойства и методы (static)**

Методы с пометкой `static` принадлежат самому классу. Они идеальны для:
-   **Фабричных методов**: создание объекта специальным способом.
-   **Утилит**: функции, связанные с сущностью, но не требующие данных конкретного объекта.

```javascript
class User {
    static role = 'user'; // Общее свойство для всех

    static createAdmin() { // Фабрика
        return new User('Admin', 'root');
    }

    static compare(userA, userB) { // Утилита
        return userA.age - userB.age;
    }
}

const admin = User.createAdmin();
```

## |3| **Геттеры и Сеттеры (get / set)**

Аксессоры позволяют превратить свойство в "умное". Это не просто чтение, а выполнение кода.

**Зачем они нужны:**
-   **Валидация**: проверка данных перед записью.
-   **Логирование**: отслеживание изменений.
-   **Вычисляемые свойства**: значение, которое зависит от других полей.

```javascript
class Temperature {
    constructor(celsius) {
        this.celsius = celsius;
    }

    get fahrenheit() {
        return this.celsius * 1.8 + 32;
    }

    set fahrenheit(value) {
        this.celsius = (value - 32) / 1.8;
    }
}
```

## |4| **Символы (Symbol) для метаданных**

**Symbol** — уникальный тип данных. В ООП они используются как "полуприватные" ключи. Они не видны в `Object.keys()` или `JSON.stringify()`, но к ним можно достучаться через специальные методы.

```javascript
const INTERNAL_ID = Symbol('id');

class Item {
    constructor(id) {
        this[INTERNAL_ID] = id;
    }
}

const item = new Item(12345);
console.log(Object.keys(item)); // symbols здесь не будет
```

## **Итог**

-   **`#`** — жесткая приватность.
-   **`static`** — логика уровня "чертежа", а не "детали".
-   **`get/set`** — контроль и валидация.
-   **`Symbol`** — уникальные ключи для скрытых метаданных.

## **Практика**

### 1. **Кофемашина v2.0**
Создайте класс `CoffeeMachine`. У неё должно быть:
- Приватное поле `#waterAmount` (сеттер не дает поставить < 0).
- Статический метод `compare`, который сравнивает две кофемашины по объему воды.
- Геттер `power`, который просто возвращает мощность (заданную один раз в конструкторе).

### 2. **Защищенный профиль**
Реализуйте класс `User`, где имя (`_name`) защищено нижним подчеркиванием. Сеттер должен проверять длину имени (минимум 3 символа) и выбрасывать ошибку `Error`, если условие не соблюдено.

### 3. **ID через Symbol**
Добавьте в класс `Task` уникальный идентификатор через `Symbol`. Напишите функцию, которая принимает объект задачи и выводит все его обычные свойства, убедившись, что ID не отображается.

### 4. **Фабрика пользователей**
Создайте класс `Account`. Добавьте статические методы `fromJSON(json)` и `anonymous()`, которые возвращают новые экземпляры класса.
