# **Прототипы и наследование**

JavaScript часто называют языком на основе прототипов. В отличие от Java или C++, здесь нет "чертежей" (классов) в классическом понимании — есть только объекты, которые делегируют свои обязанности другим объектам. В этом уроке мы разберем магию `[[Prototype]]` и то, как современные классы делают эту магию удобной.

## |1| **Прототип объекта: __proto__ vs prototype**

Это тема, на которой спотыкаются многие. В JavaScript есть два похожих, но принципиально разных свойства:

1.  **`prototype`** — это свойство **функции** (кроме стрелочных). Оно указывает, какой прототип будет у объектов, созданных через `new этой_функцией`.
2.  **`__proto__`** — это ссылка **объекта** на его прототип (тот самый "донор").

```javascript
function User(name) {
    this.name = name;
}

const alice = new User('Alice');

// prototype у функции, __proto__ у созданного объекта
console.log(User.prototype === alice.__proto__); // true
```

---

## |2| **Алгоритм работы оператора `new`**

Когда вы пишете `new User('Alice')`, JavaScript выполняет 4 шага:

1.  **Создается пустой объект** `{}`.
2.  **Устанавливается прототип**: свойство `__proto__` нового объекта связывается с `User.prototype`.
3.  **Выполняется конструктор**: функция вызывается с `this`, который теперь указывает на наш новый объект.
4.  **Возврат**: если функция не возвращает другой объект явно, возвращается созданный на шаге 1 объект.

### **Ручная эмуляция `new`**
```javascript
function customNew(Constructor, ...args) {
    const obj = Object.create(Constructor.prototype); // Шаги 1 и 2
    const result = Constructor.apply(obj, args);      // Шаг 3
    return result instanceof Object ? result : obj;   // Шаг 4
}
```

## |3| **Цепочка прототипов и Object.prototype**

Почти всё в JS наследует от `Object.prototype`. Это "верхушка" мира.

-   **Массивы**: `arr` → `Array.prototype` → `Object.prototype` → `null`.
-   **Функции**: `func` → `Function.prototype` → `Object.prototype` → `null`.

### **Где прототипа НЕТ?**
1.  У `null` и `undefined`.
2.  У объектов, созданных через `Object.create(null)` (так называемые "чистые" словари).

## |4| **Классы ES6: Краткий синтаксис**

Классы — это "синтаксический сахар" над прототипами. Они делают код чище, автоматически переводя методы в `prototype`.

```javascript
class User {
    constructor(name) {
        this.name = name;
    }

    sayHi() { // Попадает в User.prototype
        console.log(`Привет, я ${this.name}`);
    }
}
```

## |5| **Наследование (extends & super)**

Для расширения функционала используется `extends`.

```javascript
class Animal {
    constructor(name) {
        this.name = name;
    }
    eat() { console.log(`${this.name} ест`); }
}

class Rabbit extends Animal {
    hide() { console.log(`${this.name} прячется`); }
}
```

> [!CAUTION]
> Если в дочернем классе есть свой `constructor`, вы **обязаны** вызвать `super()` перед тем, как трогать `this`. В противном случае `this` не будет инициализирован.

---

## **Итог**

-   **`prototype`** — чертёж в функции. **`__proto__`** — ссылка на этот чертёж в объекте.
-   **`new`** связывает объект с прототипом и наполняет его через `this`.
-   **`Object.create(null)`** позволяет выйти из цепочки наследования.
