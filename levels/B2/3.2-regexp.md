# **Регулярные выражения (RegExp): Основы**

**Регулярные выражения** (Regular Expressions или RegExp) — это мощный инструмент для поиска, сопоставления и манипуляции строками по заданным шаблонам. Они позволяют одной строкой кода выполнить сложнейшие проверки, на которые иначе потребовались бы десятки условий `if`.

## |1| **Создание регулярного выражения**

В JavaScript есть два способа создать объект `RegExp`:

1.  **Литеральный синтаксис** (используется чаще всего):
    ```javascript
    const regex = /шаблон/флаги;
    ```
2.  **Конструктор `new RegExp`** (нужен, когда шаблон формируется динамически из строки):
    ```javascript
    const pattern = "abc";
    const regex = new RegExp(pattern, "gi");
    ```

---

## |2| **Флаги**

Флаги меняют поведение поиска:
-   `i` (ignore case): поиск без учета регистра.
-   `g` (global): поиск всех совпадений, а не только первого.
-   `m` (multiline): многострочный режим (влияет на якоря `^` и `$`).
-   `u` (unicode): полная поддержка Юникода.

---

## |3| **Основные методы**

### **Для RegExp объектов:**
-   **`regex.test(str)`**: Возвращает `true`, если совпадение найдено.
-   **`regex.exec(str)`**: Возвращает массив с деталями совпадения (или `null`).

### **Для String объектов:**
-   **`str.match(regex)`**: Возвращает массив всех совпадений (если есть флаг `g`) или детали первого.
-   **`str.replace(regex, replacement)`**: Ищет по шаблону и заменяет его.

```javascript
const str = "Я люблю JavaScript";
const regex = /javascript/i;

console.log(regex.test(str)); // true
console.log(str.replace(/люблю/, "обожаю")); // "Я обожаю JavaScript"
```

## |4| **Символьные классы и квантификаторы**

Это "кирпичики" шаблонов:
-   `\d` — цифра.
-   `\D` — не цифра.
-   `\w` — буква, цифра или подчеркивание.
-   `\s` — пробельный символ (пробел, табуляция).
-   `.` — любой символ, кроме переноса строки.

**Квантификаторы** задают количество:
-   `+` — один или более.
-   `*` — ноль или более.
-   `?` — ноль или один.
-   `{n}` — ровно `n` раз.

```javascript
const phone = "8-900-123-45-67";
const isPhone = /\d-\d{3}-\d{3}-\d{2}-\d{2}/.test(phone);
console.log(isPhone); // true
```

---

## |5| **Якоря и границы**

Позволяют искать в начале или конце строки:
-   `^` — начало строки.
-   `$` — конец строки.
-   `\b` — граница слова.

```javascript
const str = "JavaScript — это круто";
console.log(/^Java/.test(str)); // true (начинается с Java)
console.log(/круто$/.test(str)); // true (заканчивается на круто)
```

---

## **Итог**

-   Регулярные выражения — это объекты типа `RegExp`.
-   Используйте литералы `/.../` для статических шаблонов и `new RegExp()` для динамических.
-   Флаг `g` критически важен, если нужно найти или заменить *все* вхождения.
-   Всегда проверяйте регулярные выражения на граничных случаях (пустые строки, некорректные символы).

## **Практика**

### 1. **Проверка Email**
Напишите регулярное выражение для базовой проверки email (наличие `@` и точки после него). Проверьте его методом `.test()`.

### 2. **Поиск всех чисел**
Дана строка `"В 2023 году мне исполнилось 25 лет"`. С помощью метода `.match()` и регулярного выражения найдите все числа в этой строке и выведите их массивом.

### 3. **Очистка строки**
Напишите функцию `cleanString(str)`, которая удаляет из строки все символы, кроме букв и пробелов, используя `.replace()`.
