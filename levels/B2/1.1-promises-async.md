# **Промисы и асинхронные функции**

Асинхронность — это сердце современного JavaScript. Без неё браузер замирал бы при каждом запросе к серверу, а Node.js не смог бы обрабатывать тысячи подключений. В этом уроке мы разберем главный механизм управления временем в JS — Промисы.

## |1| **Что такое Промис?**

**Промис (Promise)** — это объект-обещание. Он представляет результат асинхронной операции, который будет готов "когда-нибудь".

Представьте, что вы заказали еду. Вам дали пейджер (Промис).
1.  **Pending** (Ожидание): Еда готовится.
2.  **Fulfilled** (Выполнено): Еда готова, пейджер запищал.
3.  **Rejected** (Отклонено): Продукты закончились, еды не будет.

```javascript
const myPromise = new Promise((resolve, reject) => {
    setTimeout(() => {
        const ok = true;
        if (ok) resolve("Данные получены!");
        else reject("Ошибка сети");
    }, 1000);
});
```

---

## |2| **Состояния и методы (.then, .catch, .finally)**

Промис "неизменяем" после того, как он перешел в состояние Fulfilled или Rejected.

-   **`.then(cb)`** — срабатывает при успехе.
-   **`.catch(cb)`** — срабатывает при ошибке.
-   **`.finally(cb)`** — срабатывает в любом случае (удобно для закрытия индикаторов загрузки).

```javascript
myPromise
    .then(data => console.log(data))
    .catch(err => console.error(err))
    .finally(() => console.log("Запрос завершен"));
```

---

## |3| **Цепочки промисов**

Главная сила промисов — в их способности строить цепочки. Каждый `.then` возвращает новый промис.

```javascript
fetchUser(1)
    .then(user => fetchPosts(user.id))
    .then(posts => fetchComments(posts[0].id))
    .then(comments => console.log(comments))
    .catch(err => console.log("Что-то пошло не так на любом этапе"));
```

---

## |4| **Статические методы (all, race, any)**

Иногда нужно запустить несколько операций одновременно.

1.  **`Promise.all([p1, p2])`** — Ждет ВСЕХ. Если хоть один упадет — падает всё.
2.  **`Promise.allSettled([p1, p2])`** — Ждет всех, возвращает массив статусов (успех/провал).
3.  **`Promise.race([p1, p2])`** — Кто первый пришел, тот и победил.
4.  **`Promise.any([p1, p2])`** — Ждет первого УСПЕШНОГО.

---

## |5| **Async / Await: Синтаксический сахар**

Это способ писать асинхронный код так, будто он синхронный. Под капотом это всё те же Промисы.

```javascript
async function showProfile() {
    try {
        const user = await fetchUser(1); // Ждем здесь
        const avatar = await fetchAvatar(user);
        console.log(avatar);
    } catch (err) {
        console.log("Ошибка!", err);
    }
}
```
> [!IMPORTANT]
> Ключевое слово `await` можно использовать только внутри функций, помеченных как `async` (или на верхнем уровне в современных модулях).

---

## **Итог**

-   Промисы решают проблему **Callback Hell**.
-   У промиса 3 состояния: **pending**, **fulfilled**, **rejected**.
-   `async/await` делает код чище и понятнее.
-   `try...catch` — стандарт обработки ошибок в асинхронных функциях.

## **Практика**

### 1. **Таймер-промис**
Напишите функцию `delay(ms)`, которая возвращает промис, переходящий в состояние "выполнено" через `ms` миллисекунд.
Использование: `delay(2000).then(() => alert('Прошло 2 секунды'))`.

### 2. **Гонка запросов**
Создайте два промиса: один выполняется за 1 секунду, другой за 3. Используйте `Promise.race`, чтобы определить "победителя" и вывести его результат в консоль.

### 3. **Последовательный Fetch**
Используя `async/await`, напишите функцию, которая сначала загружает пользователя по ID, а затем загружает список его друзей (используйте `https://jsonplaceholder.typicode.com/users/1` для теста).

### 4. **Умный Retry**
Напишите функцию `smartFetch`, которая пытается сделать запрос к API. Если запрос упал, она должна повторить его еще 2 раза с паузой в 1 секунду перед тем, как окончательно выбросить ошибку.
