# **Асинхронные циклы и ошибки**

В реальных приложениях асинхронные операции часто происходят пачками (списки файлов, очереди сообщений). Простого `await` бывает недостаточно, а ошибки в асинхронном коде могут обрушить всё приложение, если их не ловить правильно.

## |1| **Асинхронные циклы (for await...of)**

Обычный цикл `for...of` не умеет ждать данные из асинхронных источников (например, из потока чтения файлов). Для этого существует специальный синтаксис.

```javascript
async function processStream(stream) {
    for await (const chunk of stream) {
        console.log("Получен кусок данных:", chunk);
    }
}
```
Этот цикл итерируется по объектам, которые реализуют протокол `Symbol.asyncIterator`.

## |2| **Параллельно vs Последовательно**

Это самая частая ошибка новичков в `async/await`.

### **1. Последовательно (Медленно)**
```javascript
for (const url of urls) {
    const data = await fetch(url); // Ждем завершения каждого, прежде чем начать следующий
}
```
Цикл `for...of` (а также обычный `for` и `while`) приостанавливает выполнение на каждой итерации до завершения `await`. Это идеально, когда порядок критичен.

### **2. Ловушка `forEach`**
Метод `forEach` **не ожидает** завершения асинхронных функций.

```javascript
urls.forEach(async (url) => {
    await fetch(url); // Все запросы запустятся ОДНОВРЕМЕННО
});
console.log('Готово!'); // Это выведется ДО того, как запросы завершатся
```
Это происходит потому, что `forEach` просто вызывает колбэк и не смотрит, вернул ли он промис.

### **3. Параллельно (Быстро)**
Для параллельного выполнения с ожиданием финиша используйте `map` + `Promise.all`.

```javascript
const promises = urls.map(url => fetch(url));
const results = await Promise.all(promises); // Все запросы летят одновременно
```

### **4. Динамические условия (`while`)**
Цикл `while` полезен, когда количество итераций неизвестно (например, опрос сервера — polling).

```javascript
async function pollServer() {
    let result = null;
    while (!result) {
        const response = await fetch('/status');
        const data = await response.json();
        
        if (data.status === 'ready') result = data;
        else await new Promise(r => setTimeout(r, 1000)); // Пауза перед следующей попыткой
    }
}
```

## |3| **Генераторы и асинхронность**

Асинхронные генераторы позволяют "производить" данные со временем.

```javascript
async function* range(start, end) {
    for (let i = start; i <= end; i++) {
        await new Promise(r => setTimeout(r, 1000));
        yield i;
    }
}

// Использование:
(async () => {
    for await (const num of range(1, 5)) {
        console.log(num); // Выводит числа раз в секунду
    }
})();
```

## |4| **Кастомные классы ошибок**

В больших проектах удобно разделять ошибки (например, `NetworkError`, `ValidationError`). Это делается через наследование встроенного класса `Error`.

```javascript
class HttpError extends Error {
    constructor(status, message) {
        super(message);
        this.name = "HttpError";
        this.status = status;
    }
}

async function loadData() {
    throw new HttpError(404, "Страница не найдена");
}

try {
    await loadData();
} catch (err) {
    if (err instanceof HttpError) {
        console.error(`Ошибка сервера ${err.status}: ${err.message}`);
    } else {
        throw err; // Пробрасываем неизвестную ошибку дальше
    }
}
```

## **Итог**

-   Используйте `for await...of` для работы с асинхронными итераторами (потоки, очереди).
-   `map` + `Promise.all` — для параллельной загрузки данных.
-   `await` внутри обычного `for` — для последовательной обработки (если порядок важен).
-   Создавайте свои классы ошибок для более точного контроля в `try...catch`.

## **Практика**

### 1. **Асинхронный фильтр**
Напишите функцию `asyncFilter(array, predicate)`, где `predicate` — асинхронная проверка. Она должна возвращать новый массив только с теми элементами, для которых `predicate` вернул `true`.

### 2. **Параллельный пинг**
Есть массив из 5 URL-адресов. Используя `Promise.all` и `fetch`, получите заголовки всех страниц одновременно. Если хоть один запрос упал, функция должна поймать ошибку и не дать приложению упасть.

### 3. **Генератор бегущей строки**
Создайте асинхронный генератор, который принимает строку и выдает её по одному символу раз в 500мс. Выведите результат в консоль через `for await...of`.

### 4. **Многоуровневые ошибки**
Создайте базовый класс `AppError`, и два его подкласса: `AuthError` и `DatabaseError`. Напишите функцию, которая случайно имитирует одну из этих ошибок, и блок `try...catch`, который по-разному реагирует на каждый тип ошибки.
